# 上下文压缩机制 (Context Compression)

## 目标

节省Token消耗，提高长对话效率，平均节省 **22%** tokens。

## 压缩规则

| 原始内容 | 压缩方式 | 示例 |
|:---|:---|:---|
| 重复内容 | 删除冗余 | "请帮我...请帮我..." → "请帮我..." |
| 礼貌用语 | 简化为核心请求 | "不好意思打扰了，能否帮我..." → "请求：..." |
| 过长描述 | 转换为Markdown结构 | 长段落 → 结构化列表 |
| 多轮对话 | 提取摘要保存 | 历史对话 → 关键决策点摘要 |
| 代码块 | 保留核心逻辑 | 删除注释和空行 |

## 触发条件

- [ ] 对话超过 **10轮**
- [ ] 上下文超过 **5K tokens**
- [ ] 用户明确要求压缩
- [ ] 切换会话时

## 压缩流程

```
检测到触发条件
    ↓
分析上下文内容
    ↓
应用压缩规则
    ↓
提取关键信息到 MEMORY.md
    ↓
生成压缩摘要
    ↓
继续对话（基于摘要）
```

## 保留内容

以下内容**不压缩**：
- 关键决策和结论
- 用户明确强调的信息
- 代码和配置
- 数据和分析结果

## 压缩示例

**压缩前：**
```
用户：你好，我想请你帮我一个忙。我最近在学习Python，
但是遇到了一些问题。我想写一个程序来处理一些数据，
但是我不太清楚怎么开始。你能帮我吗？

助手：当然可以帮助你！请问你想处理什么类型的数据？

用户：谢谢你！我想处理一些CSV文件，需要读取数据然后
做一些统计分析...
```

**压缩后：**
```
📋 会话摘要
- 用户学习Python，需处理CSV数据
- 需求：数据读取 + 统计分析
- 当前：讨论具体实现方案
```

## 自动化

建议集成到Heartbeat任务，每30分钟检查一次是否需要压缩。
