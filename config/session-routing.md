# 会话识别规则 (Session Routing)

## 两步识别流程

### 第一步：上下文关联度检查

**判断标准：**
- 话题是否相关
- 是否同一任务
- 时间间隔 < 5分钟

**结果：**
- 相关度高 → 保持现有会话和模型池
- 相关度低 → 开启新会话

### 第二步：任务分类与模型池选择

**识别流程：**
```
新任务到达
    ↓
检查任务类型关键词
    ↓
匹配模型池
    ↓
输出："当前任务属于XXX，使用XX模型池"
```

## 关键词匹配规则

| 任务类型 | 关键词 | 模型池 | 说明 |
|:---|:---|:---:|:---|
| 日常会话 | 快速、简单、闲聊、查询、你好 | fast | 快速响应 |
| 复杂推理 | 分析、推理、编程、代码、架构 | smart | 深度思考 |
| 文档处理 | 文档、长文本、写作、总结 | text | 长上下文 |
| 视觉任务 | 图片、视频、看图、生成图像 | vision | 多模态 |

## 会话保持规则

**保持当前会话：**
- 话题连续
- 同一任务的不同步骤
- 追问/补充说明

**开启新会话：**
- 话题完全切换
- 时间间隔 > 10分钟
- 明确的新任务指令

## 模型切换提示

当自动切换模型池时，输出格式：
```
🔄 会话识别：当前任务属于【任务类型】
🧠 自动切换至【模型池名称】以获得最佳效果
```
