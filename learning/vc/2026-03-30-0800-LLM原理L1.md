## 一、为什么需要注意力机制？

### 1.1 RNN 的工作方式与天然缺陷

传统深度学习处理序列（如文本）用的是 **RNN（循环神经网络）**。它的结构是这样的：

```
输入: "The cat sat on the mat"

Step 1: 处理 "The"  →  hidden state h1
Step 2: 处理 "cat"  →  h2 = f(h1, "cat")    ← 必须等 h1 算完
Step 3: 处理 "sat"  →  h3 = f(h2, "sat")    ← 必须等 h2 算完
Step 4: 处理 "on"   →  h4 = f(h3, "on")     ← 必须等 h3 算完
...以此类推
```

**三个致命缺陷：**

**缺陷1：无法并行**
- 每一步都依赖前一步的 hidden state
- GPU/TPU 并行优势完全浪费
- 训练速度极慢（175B 模型用 RNN 训练需要几十年）

**缺陷2：梯度消失/爆炸**
- 反向传播时，梯度要回传很多层
- 链式法则连乘 → 梯度指数级缩小（消失）或爆炸
- 长句子前后的词几乎没有联系

**缺陷3：长距离依赖困难**
- "The **cat** that I saw yesterday **sat** on the mat"
- "cat" 和 "sat" 之间隔了 10 个词
- RNN 需要走过 10 步才能建立联系，每一步都在丢失信息

## 二、注意力机制的本质思想（2014）

### 2.1 一个生活化的例子

想象你在翻译一句话：

**英文**: "The **food** that I ordered yesterday was extremely **delicious**"
**中文**: "我昨天点的**食物****非常好吃**"

翻译"delicious"时，人类会自然地"注意"到它和"food"强相关，尽管中间隔了很多词。

**注意力机制**做的就是这件事：
> **在翻译每一个词的时候，允许模型"回头看"源句子的所有词，然后根据相关性决定关注哪些。**

### 2.2 早期注意力机制（Bahdanau Attention, 2014）

在神经机器翻译（NMT）中，解码器（生成翻译的RNN）每一步可以：
1. 看过**整个**源句子的所有 hidden states
2. 给每个词打一个**注意力权重**（0~1之间，相加=1）
3. 加权求和得到"上下文向量"
4. 结合当前解码状态，生成下一个词

```
"delicious" 的注意力权重可能是：
  The(0.02) food(0.35) that(0.01) I(0.01) ordered(0.03) 
  yesterday(0.04) was(0.02) extremely(0.04) delicious(0.48)

→ 权重大的词（food=0.35）对翻译"delicious"的帮助最大
```

**关键洞察**：这解决了 RNN 的长距离依赖问题——不管距离多远，一步就看完全部。

## 三、Query / Key / Value 三角关系

这是注意力机制最核心的数学概念。先用**查字典**来类比：

### 3.1 查字典类比

| 角色 | 现实类比 | 数学意义 |
|:---:|:---|:---|
| **Query（查询）** | 你想查的词 | "我想知道...的信息" |
| **Key（键）** | 字典里每个词的标签 | 每个位置的特征 |
| **Value（值）** | 词的解释内容 | 每个位置的实际信息 |

**查字典过程**：
```
Query: "苹果"（你想查水果）
Keys:  ["苹果手机", "红苹果", "苹果手表"]
         ↓            ↓           ↓
       0.2           0.7        0.1    ← 相关性分数（softmax后）

Value: ["苹果公司", "红色的水果", "苹果公司产品"]
加权求和: 0.2×"苹果公司" + 0.7×"红色的水果" + 0.1×"苹果公司产品"
        ≈ "红色的水果"（和 Query 最相关的 Value 被突出）
```

### 3.2 自注意力中的 Q/K/V

在 **Self-Attention（自注意力）** 中，Q、K、V 都来自**同一个输入序列**：

```python
# 这是一个 token 序列的向量表示（伪代码）
# 假设有 N 个 token，每个 token 被表示为一个 d 维向量

X = [x₁, x₂, x₃, ..., x_N]  # N × d 的矩阵，N=序列长度，d=向量维度

# 通过三个不同的线性变换，生成 Q、K、V
W_Q = Linear(d, d_k)          # 可学习的权重矩阵
W_K = Linear(d, d_k)
W_V = Linear(d, d_v)

Q = X @ W_Q  # [N, d_k]
K = X @ W_K  # [N, d_k]
V = X @ W_V  # [N, d_v]

# Q 和 K 的转置做矩阵乘法 → 得到每个位置对其他位置的相关性
# shape: [N, d_k] × [d_k, N] = [N, N]
scores = Q @ K.T   # 第 i 行 = 第 i 个 token 对所有 token 的相关性分数

# softmax 把分数变成概率分布（0~1，相加=1）
weights = softmax(scores, axis=1)

# 用权重对 V 加权求和
output = weights @ V  # [N, N] × [N, d_v] = [N, d_v]
```

### 3.3 为什么要除以 √d_k？

```
Scaled Dot-Product Attention = softmax(QK^T / √d_k) · V
```

**问题**：当 d_k（向量维度）很大时，Q·K 的点积结果会很大，导致 softmax 进入**饱和区**（梯度接近0）。

**例子**：d_k = 512，Q 和 K 的每个元素均值=0，方差=1
→ Q·K 的均值=0，但方差=512
→ 大量值集中在很大的正数区域
→ softmax → one-hot（梯度几乎为0）

**解决**：除以 √d_k，将方差缩放到合理范围
- d_k = 512 → √512 ≈ 22.6
- 点积结果除以 22.6 → 方差恢复到 1
- softmax 输出平滑，梯度正常流动

## 四、Transformer 的架构

### 4.1 整体结构（编码器-解码器）

```
                    Transformer
          ┌──────────────────────────────┐
          │         编码器 (Encoder)       │
          │  Input → N×[Multi-Head Attn]  │
          │      → [FFN] → Output        │
          └──────────────────────────────┘
                    ↓ (encoded)
          ┌──────────────────────────────┐
          │         解码器 (Decoder)       │
          │  Output → N×[Masked Attn]    │
          │      → [Cross Attn] → [FFN]  │
          │      → Linear → Softmax      │
          └──────────────────────────────┘
                    ↓
               Next Token
```

### 4.2 编码器详解

每个编码器层有两层子网络：

**① Multi-Head Self-Attention（多头自注意力）**
- "多头"：不是做一次注意力，而是做 H 次（通常 H=8）
- 每个头独立学习不同的注意力模式
- 例子：头1可能关注语法关系，头2可能关注指代关系，头3可能关注语义相似度
- 最后把 H 个头的输出拼接起来，再做一个线性变换

**② Feed-Forward Network（前馈网络）**
```python
FFN(x) = ReLU(x @ W₁ + b₁) @ W₂ + b₂
# 两层线性变换，中间加 ReLU
# 这部分和注意力无关，是纯粹的非线性变换
```

**两个"加法+归一化"残差连接**：
```python
x = LayerNorm(x + MultiHeadAttention(x))    # 注意力子层
x = LayerNorm(x + FFN(x))                     # FFN子层
```
- **残差连接**（Skip Connection）：让梯度直接传回去，避免深层网络梯度消失
- **LayerNorm**：对每个样本的每个特征做归一化，稳定训练

### 4.3 解码器与 Masked Attention

解码器多了一个 **Masked（带掩码的）自注意力**。

**为什么要 Mask？**
- 解码器是自回归生成：逐个 token 生成
- 生成第 5 个 token 时，不应该看到第 6、7、... 个 token（还没生成）
- Mask 的作用是把"未来信息"遮住

```
真实注意力矩阵（可以看到所有位置）：
         The  cat  sat
The      1    0    0
cat      1    1    0
sat      1    1    1

带 Mask 的注意力矩阵（只能看当前位置及之前）：
         The  cat  sat
The      1    0    0    ← 只能看 The
cat      1    1    0    ← 能看 The, cat
sat      1    1    1    ← 能看 The, cat, sat
```

## 五、位置编码：Attention 不需要位置信息？

### 5.1 问题

Self-Attention 是**置换不变**的：
```
Attention("The cat") == Attention("cat The")
```
调换 token 顺序，注意力矩阵完全不变！对序列任务来说这是灾难性的。

### 5.2 解决方案：位置编码（Positional Encoding）

给每个位置注入一个独特的"位置信号"：

```python
# 论文中使用正弦/余弦函数
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
# pos = 位置（0, 1, 2, ...）
# i   = 维度索引（0 ~ d_model-1）
```

**为什么用正弦余弦？**
- 每个位置的编码是唯一的
- 相对位置可以通过线性变换得到：PE(pos+k) 可以表示为 PE(pos) 的线性函数
- 模型可以学到"位置 i 和位置 i+3 的关系"这种相对位置

后续模型改进了位置编码：**RoPE**（旋转位置编码，LLaMA 使用）和 **ALiBi**（不学位置编码，用注意力偏置）。

## 六、完整注意力计算流程

```python
import torch
import torch.nn.functional as F
import math

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q, K, V: [batch, heads, seq_len, d_k]
    mask:    [batch, 1, seq_len, seq_len] or [batch, 1, 1, seq_len]
    """
    d_k = Q.shape[-1]

    # 1. 计算点积注意力分数
    scores = torch.matmul(Q, K.transpose(-2, -1))  # [B, H, L, L]

    # 2. 缩放
    scores = scores / math.sqrt(d_k)

    # 3. 应用 mask（解码器用，把未来位置填 -inf）
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))

    # 4. softmax 归一化
    weights = F.softmax(scores, dim=-1)  # 每行相加=1

    # 5. 对 V 加权求和
    output = torch.matmul(weights, V)    # [B, H, L, d_v]

    return output, weights
```

## 七、常见误区

| 误区 | 正确理解 |
|:---|:---|
| "Attention 就是加权平均" | 对，但权重是 Query-Key 相关性学习出来的 |
| "Self-Attention 解决了 RNN 的并行问题" | ✅ 是的，这是它最核心的价值 |
| "Attention 不需要位置信息" | 错！必须加位置编码，否则顺序信息丢失 |
| "多头 Attention 就是做多次 Attention 取平均" | 不对，是拼接后投影，每个头学到不同模式 |
| "解码器可以看到未来信息" | 错，Masked Attention 确保看不到 |

## 八、今天的核心知识点

1. **RNN 的三个缺陷**：无法并行、梯度消失/爆炸、长距离依赖弱
2. **注意力机制的直觉**：在生成每个词时，动态决定应该"关注"源句子的哪些词
3. **Query/Key/Value 三角**：
   - Query = 我想查找什么
   - Key = 每个位置的身份标识
   - Value = 每个位置的实际内容
4. **Scaled Dot-Product**：除以 √d_k 防止 softmax 饱和
5. **Transformer = Multi-Head Attention + FFN + 残差 + LayerNorm + 位置编码**
6. **Masked Attention**：解码器不能看到未来 token
