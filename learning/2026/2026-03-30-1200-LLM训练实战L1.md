# LLM训练实战 第1节 | 2026-03-30 12:00

## 今日目标
理解分布式训练的三大范式，能够判断不同并行策略的适用场景。

## 核心内容

### 为什么 LLM 必须分布式训练
- GPT-3 175B 参数：FP16 也需要 350GB 显存
- 单卡 A100 80GB，无法装下175B模型
- 必须跨 GPU 甚至跨节点分布

### 范式一：数据并行（Data Parallelism）
- 每个 GPU 有完整模型副本，不同 batch 数据分布处理
- 通信：AllReduce 同步梯度
- 优点：扩展简单，N 张卡 ≈ N 倍吞吐
- 缺点：显存不节省

### 范式二：模型并行（Tensor/Pipeline Parallelism）
- 张量并行（Tensor Parallelism）：单层横切分布到多卡
- 流水线并行（Pipeline Parallelism）：不同层分布到不同 GPU
- 通信：集合通信（AllReduce/AllGather）

### 范式三：专家并行（Expert Parallelism / MoE）
- 多个"专家"网络，每次只激活部分专家
- 结合数据并行：top-k 专家路由

## 重点概念
- **数据并行**：显存不省，吞吐翻倍（适合显存够大的场景）
- **张量并行**：单层跨卡，需要 NVLink（低延迟）
- **流水线并行**：通信量小，适合跨节点
