**KV Cache：为何需要/显存占用计算/加速Decode的原理**

---

## 📖 完整讲解

## 原理

生成式模型的痛点：生成下一个 token 时，朴素实现需要重新计算所有历史 token 的注意力。

KV Cache 把已计算的 Key 和 Value 缓存起来，每次只算新 token 的注意力。

```python
cache_k, cache_v = [], []
for new_token in generated_tokens:
    q = project_q(new_token)
    k = project_k(new_token)
    v = project_v(new_token)
    k_full = torch.cat([cache_k, k], dim=-2)
    v_full = torch.cat([cache_v, v], dim=-2)
    attn = attention(q, k_full, v_full)
    cache_k, cache_v = k_full, v_full
```

## 显存计算

13B 模型 FP16：每个 token 的 K/V 约 1.6MB。

4096 上下文：约 6.6GB。70B 模型 4096 上下文约 33GB。

vLLM 的 PagedAttention 把 KV Cache 分成 4KB page 管理，显存利用率从 30% 提升到 90%+。
