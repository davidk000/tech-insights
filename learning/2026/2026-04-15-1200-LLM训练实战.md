## 📖 完整讲解

## 原理

Attention 本身是顺序无关的——把输入打乱，注意力分数不变。位置编码（PE）给模型注入序列顺序信息。

### 三大方案对比

| 方案 | 公式 | 优点 | 缺点 |
|:---|:---|:---|:---|
| Sinusoidal | PE(pos,i)=sin(pos/10000^(i/d)) | 无需学习，可外推 | 表达能力有限 |
| **RoPE** | 旋转矩阵 | 保留相对位置，SOTA | 实现复杂 |
| ALiBi | 线性偏置 | 无需编码，外推强 | 部分任务略降 |

**RoPE（Rotary Position Embedding）** 是当前主流，LLaMA、GLM、Qwen 均采用。核心：不对 Q/K 做加法，而是旋转——位置信息编码在旋转矩阵里，内置保留相对位置。

直觉：`R_m·q` 和 `R_n·k` 的点积只和相对位置 (m-n) 有关，与绝对位置无关。

```python
def apply_rotary_emb(x, cos, sin):
    x1, x2 = x[..., :x.shape[-1]//2], x[..., x.shape[-1]//2:]
    return torch.cat([x1*cos - x2*sin, x1*sin + x2*cos], dim=-1)
```

## 常见误区

1. "PE 加在 Embedding 上就够了" → 外推到训练没见过长度时效果差
2. "RoPE 比 Sinusoidal 效果好很多" → 长上下文场景明显，短文本差异不大
3. "位置编码不重要" → 错。它决定模型能否区分「A 在 B 前」和「B 在 A 前」
