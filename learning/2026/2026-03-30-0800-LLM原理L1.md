# LLM原理 第1节 | 2026-03-30 08:00

## 今日目标
理解 Transformer 诞生背景，掌握注意力机制的核心思想。

## 核心内容

### RNN 的根本缺陷
- 序列处理必须逐 token 输入，无法并行
- 梯度消失/爆炸问题（BPTT 截断）
- 长距离依赖必须经过多步才能建立

### 注意力机制的核心思想（2014-NMT）
- Bahdanau Attention：Decoder 每步可"回头看"整个源序列
- Key-Value-Query 三角：Query 关注相关 Key，计算加权和得到上下文

### Transformer 论文核心贡献（Vaswani et al., 2017）
- 完全抛弃 RNN/LSTM，纯 Attention 机制
- 多头自注意力（Multi-Head Self-Attention）
- 残差连接 + LayerNorm + 位置编码
- 编码器-解码器架构

### 数学形式（注意力分数）
```
Attention(Q,K,V) = softmax(QK^T / √d_k) · V

本质：对所有 Key-Value 对做加权求和，权重由 Query 与 Key 的点积相似度决定
```

## 重点概念
- **Self-Attention**：Query/Key/Value 都来自同一个输入（Token 序列自身）
- **Scaled Dot-Product**：除以 √d_k 防止点积过大导致 softmax 梯度饱和

## 知识沉淀
用 knowledge-graph 记录：Self-Attention 是 Transformer 的核心，它让任意两个 Token 之间可以并行建立依赖关系，摆脱了 RNN 的序列化限制。
