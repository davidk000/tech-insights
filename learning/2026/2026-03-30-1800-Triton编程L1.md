## 今日目标
理解 Triton 的定位，知道它解决的核心问题。

## 核心内容

### CUDA 的痛点
- 手写 CUDA Kernel 开发门槛高：共享内存/Tiling/同步/向量化都要手工处理
- 优化CUDA代码需要深入GPU架构知识（SM/Warp/Shared Memory）
- 不同 GPU 架构（Volta/Ampere/Hopper）优化策略不同

### Triton 的核心定位
- 开源（OpenAI）：Python-like DSL，编译成高效 CUDA/ROCm
- 自动做 Tiling/Vectorization/Coalesced Memory Access
- 程序员关注算法逻辑，不用手动管内存层次

### Triton vs CUDA
```
CUDA:      写kernel → 手动Tiling → 手动SharedMem → 手动向量化
Triton:    写kernel → 自动Tiling → 自动优化 → 生成PTX/ILP
```

### Triton 适用场景
- AI 算子：Softmax/GELU/LayerNorm/Attention
- 自定义操作：PyTorch 中没有的算子
- 不适合：复杂控制流/递归/不规则内存访问

### Triton 安装
```bash
pip install triton
# 或 conda: conda install triton -c pytorch
```

## 重点概念
- **Block/Tile**：将大矩阵切分成小块并行处理
- **自动向量化**：Triton 自动用 SIMD 指令
- **自动 Tiling**：Triton 编译器决定最优分块大小
