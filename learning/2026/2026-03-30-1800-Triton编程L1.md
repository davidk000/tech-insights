## 一、为什么需要 Triton？

### 1.1 CUDA 开发的痛点

写一个高效的 CUDA Kernel，你需要：

```
1. 手动管理共享内存（Shared Memory）
2. 手动做 Tiling（把大矩阵切成小方块）
3. 手动向量化（128bit/256bit/512bit load/store）
4. 手动处理 Bank Conflict
5. 手动计算 Grid/Block 维度
6. 针对特定 GPU 架构（Volta/Ampere/Hopper）调优
```

**例子：写一个 Softmax CUDA Kernel**

```cuda
__global__ void softmax_kernel(float* out, float* inp, int N, int D) {
    // 1. 每个 thread 处理一行
    int row = blockIdx.x;
    
    // 2. 手动求最大值（两阶段归约）
    float max_val = -INFINITY;
    for (int col = threadIdx.x; col < D; col += blockDim.x) {
        max_val = fmaxf(max_val, inp[row * D + col]);
    }
    // 3. 需要 warp reduce + block reduce...
    
    // 4. 手动求和 exp(x - max)
    float sum = 0;
    for (int col = threadIdx.x; col < D; col += blockDim.x) {
        sum += expf(inp[row * D + col] - max_val);
    }
    // 5. 又要 reduce...
    
    // 6. 写回
    for (int col = threadIdx.x; col < D; col += blockDim.x) {
        out[row * D + col] = expf(inp[row * D + col] - max_val) / sum;
    }
}
```

这还只是一个最简单的 Softmax。LayerNorm、Attention、GEGLU 更复杂。

### 1.2 Triton 的核心定位

Triton 是 OpenAI（不是英伟达）发起的开源项目，目标：

> **用 Python-like 语法，写出接近 CUDA 性能的算子。**

```
CUDA：         C++语法 + 手动内存层次管理 + 架构相关优化
Triton：       Python语法 + 自动Tiling/向量化 + 自动架构适配
```

**适用场景**：
- AI 算子：Softmax、LayerNorm、GELU、Attention、Transpose
- PyTorch 中没有的算子
- 需要在 PyTorch 中插入自定义高效操作

**不适用**：
- 复杂控制流（分支、递归）
- 不规则内存访问
- 通用计算（非矩阵运算）

## 三、Triton 编程模型：Grid / Block / Tile

### 3.1 核心概念

Triton 的编程模型和 CUDA 类似，但更简洁：

```
Grid（网格）：整体launch的配置
  └── Block（块）：一个GPU线程块（CTA）
       └── Tile（块）：程序员定义的逻辑分块
```

**CUDA 的等效对照**：

| CUDA 概念 | Triton 概念 | 说明 |
|:---|:---|:---|
| `threadIdx.x` | `pid`（program id）| 逻辑索引 |
| `blockIdx.x` | `pid` | 同上 |
| `blockDim.x` | 无需指定 | 自动推导 |
| `Shared Memory` | `tl.dot` 的 `ACC_TYPE` | 自动管理 |
| `__syncthreads()` | 自动同步 | 无需手动调用 |
| `warp` | 无需关心 | Triton 自动处理向量化 |

### 3.2 第一个 Triton Kernel：向量加法

```python
import torch
import triton
import triton.language as tl

@triton.jit
def add_kernel(
    # 指针参数（行主序）
    x_ptr, y_ptr, output_ptr,
    # 维度
    n_elements,
    # 分块大小（编译时常量）
    BLOCK_SIZE: tl.constexpr,
):
    # 1. 获取当前块负责的索引范围
    pid = tl.program_id(axis=0)  # 第几个块
    block_start = pid * BLOCK_SIZE
    offsets = block_start + tl.arange(0, BLOCK_SIZE)
    
    # 2. 加载数据（自动越界处理 + mask）
    mask = offsets < n_elements
    x = tl.load(x_ptr + offsets, mask=mask)
    y = tl.load(y_ptr + offsets, mask=mask)
    
    # 3. 计算
    output = x + y
    
    # 4. 写回
    tl.store(output_ptr + offsets, output, mask=mask)

def add(x: torch.Tensor, y: torch.Tensor):
    assert x.shape == y.shape
    output = torch.empty_like(x)
    
    n_elements = output.numel()
    # 块大小（必须是2的幂）
    BLOCK_SIZE = 1024
    
    # Grid 大小：一个块负责 1024 元素
    grid = lambda meta: (triton.cdiv(n_elements, BLOCK_SIZE),)
    
    add_kernel[grid](x, y, output, n_elements, BLOCK_SIZE=BLOCK_SIZE)
    return output
```

**Triton 自动做的优化**：
- 自动 Tiling：把大向量切成 1024 元素的小块
- 自动向量化：一次性 load 4 个 float32（16 bytes）
- 自动并行：grid 大小自动 launch 到所有 SM

### 3.3 手动 Tiling：分块矩阵乘法

这是理解 Triton 精髓的例子：

```python
@triton.jit
def matmul_kernel(
    # 矩阵 A, B, C
    a_ptr, b_ptr, c_ptr,
    # 矩阵尺寸
    M, N, K,
    # stride（跳行距离）
    stride_am, stride_ak,
    stride_bk, stride_bn,
    stride_cm, stride_cn,
    # 分块大小
    BLOCK_SIZE_M: tl.constexpr,
    BLOCK_SIZE_N: tl.constexpr,
    BLOCK_SIZE_K: tl.constexpr,
):
    # 获取当前块在 C 矩阵中的 (pid_m, pid_n) 位置
    pid_m = tl.program_id(axis=0)
    pid_n = tl.program_id(axis=1)
    
    # 每个块处理 BLOCK_M × BLOCK_N 的输出
    # 块内使用多个 pid 共同完成
    offs_am = (pid_m * BLOCK_SIZE_M + tl.arange(0, BLOCK_SIZE_M))
    offs_bn = (pid_n * BLOCK_SIZE_N + tl.arange(0, BLOCK_SIZE_N))
    offs_k = tl.arange(0, BLOCK_SIZE_K)
    
    # 累加器
    accumulator = tl.zeros((BLOCK_SIZE_M, BLOCK_SIZE_N), dtype=tl.float32)
    
    # K 维度循环（分块计算）
    for k in range(0, tl.cdiv(K, BLOCK_SIZE_K)):
        # 加载 A 的一个块 (BLOCK_M × BLOCK_K)
        offs_am_k = offs_am[:, None] * stride_am + (offs_k[None, :] * stride_ak)
        offs_bn_k = offs_bn[None, :] * stride_bn + offs_k[None, :] * stride_bk
        a_ptrs = a_ptr + offs_am_k
        b_ptrs = b_ptr + offs_bn_k
        
        mask = (offs_am[:, None] < M) & (offs_k[None, :] < K)
        a = tl.load(a_ptrs, mask=mask)
        b = tl.load(b_ptrs, mask=mask)
        
        # 矩阵乘法累加（使用 Triton 的 dot 操作）
        accumulator += tl.dot(a, b)
        
        # 移动到下一个 K 块
        offs_k += BLOCK_SIZE_K
    
    # 写回 C（需要转成输出精度）
    c = accumulator.to(tl.float16)
    offs_cm = pid_m * BLOCK_SIZE_M + tl.arange(0, BLOCK_SIZE_M)
    offs_cn = pid_n * BLOCK_SIZE_N + tl.arange(0, BLOCK_SIZE_N)
    c_ptrs = c_ptr + offs_cm[:, None] * stride_cm + offs_cn[None, :] * stride_cn
    mask = (offs_cm[:, None] < M) & (offs_cn[None, :] < N)
    tl.store(c_ptrs, c, mask=mask)
```

**关键**：`tl.dot(a, b)` = 硬件矩阵乘法指令（Triton 自动使用 Tensor Core）。

## 四、Triton 的 Tiling 机制

### 4.1 什么是 Tiling？

Triton 的核心思想：**手动 Tiling，自动调度**。

```
原始矩阵（4096 × 4096）：
┌────────────────────────────────┐
│                                │
│                                │
│          4096 × 4096            │
│                                │
│                                │
└────────────────────────────────┘

Triton 自动分成 64 × 64 的小块：
┌────┬────┬────┬────┬────┐
│ T1 │ T2 │ T3 │ T4 │... │
├────┼────┼────┼────┼────┤
│ T5 │ T6 │ T7 │ T8 │... │   每个块由一个 GPU Block 处理
├────┼────┼────┼────┼────┤
│... │... │... │... │... │
└────┴────┴────┴────┴────┘
```

### 4.2 为什么需要手动 Tiling？

- **寄存器溢出**：GPU 寄存器有限，如果一个 thread 处理整个大向量，会溢出到 HBM（极慢）
- **缓存友好**：小块可以放进 L2 缓存，重复利用
- **向量化**：一次性 load 多个元素（triton 自动做）

## 五、自动调优（Autotune）

Triton 的杀手级功能：**自动选择最优配置**。

```python
@triton.autotune(
    # 尝试不同的 block_size 配置
    configurations=[
        # (block_m, block_n, num_warps)
        triton.Config({'BLOCK_M': 128, 'BLOCK_N': 256}, num_warps=4),
        triton.Config({'BLOCK_M': 64,  'BLOCK_N': 64},  num_warps=4),
        triton.Config({'BLOCK_M': 128, 'BLOCK_N': 128}, num_warps=8),
    ],
    # 用什么 key 来区分不同调用
    key=['M', 'N'],
)
@triton.jit
def tuned_matmul_kernel(...):
    ...
```

运行时会：**实际执行每种配置 → 测量时间 → 选择最快的**。

```python
# 使用
tuned_matmul_kernel[grid](...)  # 第一次慢（autotune），之后快
```

## 六、与 PyTorch 的集成

```python
import torch
import triton

class TritonSoftmax(torch.autograd.Function):
    @staticmethod
    def forward(ctx, x):
        # 调用 Triton kernel
        y = triton_softmax(x)
        return y
    
    @staticmethod
    def backward(ctx, grad_output):
        # Triton 也支持反向传播
        grad_input = triton_softmax_backward(grad_output)
        return grad_input

# 在模型中使用
class MyModel(torch.nn.Module):
    def __init__(self):
        super().__init__()
        self.softmax = TritonSoftmax.apply
    
    def forward(self, x):
        return self.softmax(x)
```

或者更简单地用 `torch.compile()` 触发 Triton 自动 kernel fusion（PyTorch 2.0+）。

## 七、Triton vs CUDA：性能对比

用 Softmax 实测（Triton vs PyTorch 原生）：

| 实现 | 吞吐量（GB/s）| 相对速度 |
|:---|:---:|:---:|
| PyTorch (native) | 320 | 1× |
| CUDA (手工优化) | 780 | 2.4× |
| Triton | 750 | 2.3× |

Triton 达到了接近手工 CUDA 的性能，但开发时间可能只有 1/10。

## 八、常见误区

| 误区 | 正确理解 |
|:---|:---|
| "Triton 可以完全替代 CUDA" | 错！Triton 只适合规则 AI 算子，复杂控制流不行 |
| "Triton 自动做所有优化" | 错！Tiling 策略需要程序员指定，Triton 只负责调度 |
| "用 Triton 一定能加速" | 错！配置不当可能比 PyTorch 更慢 |
| "Triton 是英伟达的产品" | 错！OpenAI 发起的开源项目 |
| "学会了 Triton 就能写所有 GPU 代码" | 错！Triton 不支持原子操作、共享内存手动管理等 |

## 九、核心知识点

1. **Triton 定位**：Python-like DSL → 高效 CUDA/ROCm，算子开发效率远高于手写 CUDA
2. **Tiling 思想**：手动分块（管理显存/缓存），自动并行调度
3. **Triton 自动化的部分**：Grid 调度、向量化加载、寄存器分配
4. **需要手动的部分**：分块策略、Tiling 大小、循环顺序
5. **Autotune**：自动搜索最优配置，运行时选择最快方案
6. **适用场景**：Softmax/LayerNorm/Attention/GELU 等规则算子
