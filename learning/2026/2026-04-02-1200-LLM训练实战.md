## 📌 今日课题
**DeepSpeed ZeRO：Stage1参数分片/Stage2优化器分片/Stage3全部分片**

---

## 📖 完整讲解

## 原理

大模型训练的核心问题：显存不够。

175B 参数的模型，光模型本身就要 350GB（FP16），还没算优化器状态和梯度。

ZeRO（Zero Redundancy Optimizer）的核心思路：分片而非复制。

| Stage | 分片内容 | 显存节省 |
|:---|:---|:---|
| ZeRO-1 | 优化器状态分片 | ~4× |
| ZeRO-2 | + 梯度分片 | ~8× |
| ZeRO-3 | + 参数分片 | ~N×（N=GPU数）|

## 显存计算实战

以 LLaMA-7B（FP16）为例：
- 模型参数：7B × 2B = 14GB
- 优化器（Adam）：7B × 2 × 4B = 56GB ← 主力显存消耗！
- 梯度：7B × 2B = 14GB
- **ZeRO-3 后**：每 GPU 约 `(14/N + 14 + 2)` GB

8 卡时每 GPU 约 20GB，65B 模型用 ZeRO-3 + 8×A100 可以训。

## 配置示例

```python
ds_config = {
    "zero_optimization": {"stage": 3},
    "fp16": {"enabled": True},
    "gradient_accumulation_steps": 4,
}
```

## 常见误区

1. "ZeRO-3 会严重拖慢通信" → 对，但 Gradient Checkpointing 可以节省 60% 显存
2. "ZeRO 比 DDP 慢很多" → 小模型可能更慢，大模型才是主场
3. "ZeRO-2 够用了" → 超过 13B 建议 ZeRO-3
