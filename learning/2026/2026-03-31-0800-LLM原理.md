## 一、从"加权平均"到"三步投影"

第1节我们留了一个核心公式：

$$
\text{Attention}(Q, K, V) = \softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

这一节我们就来**亲手把每个符号都敲一遍**，搞懂它为什么长这样。

## 二、Q/K/V 的数学推导

### 2.1 问题设定

有一个输入序列 $\mathbf{X} = [\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_N]$，每个 $\mathbf{x}_i \in \mathbb{R}^d$ 是$d$维向量。

**目标**：为序列中每个位置 $i$，计算一个新的向量 $\mathbf{z}_i$，让它同时包含**整个序列**的信息，而不只是第 $i$ 个词本身的信息。

这和 RNN 走一步看一步的做法正好相反——Attention 选择**一步看完全部**，然后按重要性加权。

### 2.2 三次线性投影

我们不希望直接用 $\mathbf{x}_i$ 本身来计算相关性。原因：

1. $\mathbf{x}_i$ 是 embedding，它的主要任务是**表示语义**
2. 我们需要一个**专门的"查询角色"**、一个**专门的"键角色"**、一个**专门的"值角色"**
3. 让同一个向量同时扮演三个角色，会让模型学得很拧巴

所以，用三个不同的线性变换把 $\mathbf{x}_i$ 投影到三个不同的空间：

$$
\mathbf{q}_i = \mathbf{x}_i W_Q, \quad \mathbf{k}_i = \mathbf{x}_i W_K, \quad \mathbf{v}_i = \mathbf{x}_i W_V
$$

其中：

| 矩阵 | 形状 | 作用 |
|:---|:---|:---|
| $W_Q \in \mathbb{R}^{d \times d_k}$ | 可学习 | Query 投影，把 $d$ 维映射到 $d_k$ 维 |
| $W_K \in \mathbb{R}^{d \times d_k}$ | 可学习 | Key 投影，把 $d$ 维映射到 $d_k$ 维 |
| $W_V \in \mathbb{R}^{d \times d_v}$ | 可学习 | Value 投影，把 $d$ 维映射到 $d_v$ 维 |

$d_k$ 和 $d_v$ 通常等于 $d / h$（$h$ = 头数），这样多头拼接后维度一致。

### 2.3 相关性分数：点积的物理意义

怎么衡量"第 $i$ 个词和第 $j$ 个词有多相关"？

**直觉**：把每个词看作高维空间中的一个向量。**相关性 = 方向接近程度**。方向越接近，点积越大。

$$
\text{score}_{ij} = \mathbf{q}_i \cdot \mathbf{k}_j = \sum_{t=1}^{d_k} q_{it} \cdot k_{jt}
$$

**生活类比**：你和一群人坐在一个房间里，你想找到"和我兴趣最接近的人"。
- Query = 你自己（你拿着一个问题去找人）
- Key = 每个人的自我介绍标题
- Value = 每个人的详细资料

你的 Query 和谁的 Key 点积最高，你就最应该去读谁的 Value。

### 2.4 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SimpleSelfAttention(nn.Module):
    def __init__(self, d_model, d_k, d_v):
        super().__init__()
        self.W_Q = nn.Linear(d_model, d_k, bias=False)
        self.W_K = nn.Linear(d_model, d_k, bias=False)
        self.W_V = nn.Linear(d_model, d_v, bias=False)
        # bias=False 是因为投影后不需要偏置

    def forward(self, x):
        # x: [batch, seq_len, d_model]
        Q = self.W_Q(x)  # [B, L, d_k]
        K = self.W_K(x)  # [B, L, d_k]
        V = self.W_V(x)  # [B, L, d_v]

        # 批量计算点积：第 i 个 Query 对所有 Key 的分数
        # Q @ K^T: [B, L, d_k] × [B, d_k, L] = [B, L, L]
        scores = torch.matmul(Q, K.transpose(-2, -1))
        return scores
```

### 2.5 常见误区

| 误区 | 正确理解 |
|:---|:---|
| "Q/K/V 都是从输入直接拿来的" | 错，都经过了不同的线性投影 |
| "Q 和 K 维度必须相同" | 对，$d_k$ 要一样才能做点积 |
| "点积越大，相关性越强" | 对，这是余弦相似度的未归一化版本 |
| "$W_Q, W_K, W_V$ 是固定的" | 错，**都是可学习参数**，通过反向传播更新 |

## 三、Scaled Dot-Product：为什么必须除以 $\sqrt{d_k}$

### 3.1 问题：点积会"爆炸"

当 $d_k$ 较大时，**点积的方差会随 $d_k$ 线性增长**。

数学推导（可跳过但建议理解）：

假设 $\mathbf{q}$ 和 $\mathbf{k}$ 的每个元素相互独立，均值 $= 0$，方差 $= 1$。

$$
\text{Var}(q_i \cdot k_i) = \text{Var}(q_i) \cdot \text{Var}(k_i) = 1 \cdot 1 = 1
$$

由于点积是 $d_k$ 个独立乘积之和：

$$
\text{Var}\left(\sum_{i=1}^{d_k} q_i k_i\right) = \sum_{i=1}^{d_k} \text{Var}(q_i k_i) = d_k \cdot 1 = d_k
$$

**当 $d_k = 512$ 时，点积方差是 512**。这意味着大多数值集中在很大的正数区域。

### 3.2 softmax 的饱和问题

```python
import torch
import torch.nn.functional as F

# 模拟大方差输入
large_logits = torch.tensor([5.0, 6.0, 7.0, 8.0])  # 方差大
small_logits = torch.tensor([0.5, 0.6, 0.7, 0.8])  # 方差小

print(F.softmax(large_logits, dim=-1))
# tensor([0.0119, 0.0321, 0.0871, 0.8689])  ← 接近 one-hot，梯度几乎为 0

print(F.softmax(small_logits, dim=-1))
# tensor([0.2170, 0.2380, 0.2596, 0.2853])  ← 平滑分布，各位置都有梯度
```

当 logits 方差很大时，softmax 输出趋向于 **one-hot**（最大项接近 1，其余接近 0）。此时梯度趋近于 0，模型几乎学不到东西。

### 3.3 解决方案：缩放

$$
\boxed{\text{scores} = \frac{\mathbf{q}_i \cdot \mathbf{k}_j}{\sqrt{d_k}}}
$$

除以 $\sqrt{d_k}$ 后，点积方差恢复到 $1$，softmax 输出平滑。

**为什么是 $\sqrt{d_k}$ 而不是 $d_k$？**

因为点积的方差是 $d_k$，所以 $\sqrt{d_k}$ 归一化后方差为：

$$
\text{Var}\left(\frac{q \cdot k}{\sqrt{d_k}}\right) = \frac{d_k}{d_k} = 1
$$

刚刚好。

### 3.4 完整 Scaled Dot-Product 实现

```python
def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q: [B, H, L, d_k]  L = seq_len
    K: [B, H, L, d_k]
    V: [B, H, L, d_v]
    mask: [B, H, L, L] or [B, 1, L, L]（广播）
    """
    d_k = Q.shape[-1]

    # Step 1: 计算点积 [B, H, L, L]
    scores = torch.matmul(Q, K.transpose(-2, -1))

    # Step 2: 缩放
    scores = scores / math.sqrt(d_k)

    # Step 3: Mask（Masked Attention 在这里处理）
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))

    # Step 4: Softmax → 注意力权重
    weights = F.softmax(scores, dim=-1)  # 每行相加 = 1

    # Step 5: 加权求和
    output = torch.matmul(weights, V)  # [B, H, L, d_v]

    return output, weights
```

### 3.5 常见误区

| 误区 | 正确理解 |
|:---|:---|
| "除以 $\sqrt{d_k}$ 是随意选的" | 不是，是为了让点积方差归一化到 1 |
| "Scaled 后的值越小越好" | 错，目的是让 softmax 输出平滑，梯度流动正常 |
| "不缩放模型也能训，只是慢一点" | 错，$d_k$ 大时几乎无法收敛，梯度消失 |

## 四、Multi-Head Attention：八个头各学什么

### 4.1 为什么需要"多头"

单头 Attention 的问题：**每个头只能学出一种相关性模式**。

现实任务中，词语之间的关系是多元的：
- 头1：学习**语法主谓关系**（"狗"关注"叫"）
- 头2：学习**指代关系**（"它"关注"狗"）
- 头3：学习**语义相似度**（"狗"和"犬"）
- 头4：学习**位置邻近关系**（相邻词之间的依赖）

单头注意力相当于"用同一个尺子量所有关系"，必然丢失信息。

### 4.2 数学公式

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O
$$

其中每个头独立计算：

$$
\text{head}_i = \text{Attention}(Q W_Q^i, K W_K^i, V W_V^i)
$$

最终输出 $W^O \in \mathbb{R}^{h d_v \times d_{\text{model}}}$ 把拼接后的向量投影回 $d_{\text{model}}$ 维度。

### 4.3 PyTorch 实现

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        self.h = num_heads
        self.d_k = d_model // num_heads  # 每个头的维度
        self.d_v = self.d_k               # 通常 d_k = d_v

        # 每个头有自己的 W_Q, W_K, W_V（实际上用一个大矩阵一次性算）
        self.W_Q = nn.Linear(d_model, d_model, bias=False)
        self.W_K = nn.Linear(d_model, d_model, bias=False)
        self.W_V = nn.Linear(d_model, d_model, bias=False)
        self.W_O = nn.Linear(d_model, d_model, bias=False)

    def forward(self, Q, K, V, mask=None):
        batch = Q.size(0)

        # 线性投影后，分成多个头
        Q = self.W_Q(Q).view(batch, -1, self.h, self.d_k).transpose(1, 2)
        K = self.W_K(K).view(batch, -1, self.h, self.d_k).transpose(1, 2)
        V = self.W_V(V).view(batch, -1, self.h, self.d_v).transpose(1, 2)

        # 调用上一节的 scaled dot-product attention
        attn_output, attn_weights = scaled_dot_product_attention(Q, K, V, mask)

        # 拼接多个头: [B, H, L, d_k] → [B, L, H*d_k] = [B, L, d_model]
        attn_output = attn_output.transpose(1, 2).contiguous()
        attn_output = attn_output.view(batch, -1, self.h * self.d_v)

        # 最终线性投影
        output = self.W_O(attn_output)
        return output, attn_weights
```

### 4.4 常见误区

| 误区 | 正确理解 |
|:---|:---|
| "多头就是多做几次 Attention 然后平均" | 错，是拼接后投影，不同头学到的模式不同 |
| "头数越多越好" | 不是，$h$ 是超参数，通常 $h=8$（GPT-3 同款配置） |
| "每个头的 $W_Q, W_K, W_V$ 随机初始化后就固定了" | 错，和其他权重一起通过反向传播学习 |
| "所有头学到的模式可以互相替代" | 错，注意力头具有**专家分工**特性，可解释性研究证实了这一点 |

## 五、Masked Attention：解码器如何"不看未来"

### 5.1 问题

解码器在生成第 $t$ 个 token 时：
- **应该看到**：位置 $1, 2, \ldots, t$（已经生成的词 + 起始符）
- **不应该看到**：位置 $t+1, t+2, \ldots, L$（还没生成的词）

如果不做 Mask，模型在训练时可以看到"标准答案"，相当于**考试作弊**，推理时根本没有这个信息。

### 5.2 Mask 的实现方式

Mask 的核心思想：用 $-\infty$ 填充那些"不该看"的位置，softmax 后自然变成 0。

```python
def create_causal_mask(seq_len, device):
    """
    生成下三角 Mask（Current Position 及之前为 1，之后为 0）
    shape: [seq_len, seq_len]
    """
    # [seq_len, seq_len] 下三角为 True（保留），上三角为 False（Mask）
    mask = torch.tril(torch.ones(seq_len, seq_len, device=device))
    return mask  # 0 = mask掉, 1 = 保留

def create_causal_mask_lookahead(seq_len, device):
    """
    更精确的版本：允许当前 token 看到自己，但看不到之后的位置
    """
    mask = torch.tril(torch.ones(seq_len, seq_len, device=device))
    return mask

# 在 Attention 中使用：
scores = scores.masked_fill(mask == 0, float('-inf'))
weights = F.softmax(scores, dim=-1)
```


### 5.3 Padding Mask（另一个必须知道的 Mask）

真实输入 batch 中，不同样本序列长度不同，需要 padding 到统一长度。padding 位置不应该参与注意力计算。

```python
def create_padding_mask(seq, pad_idx=0):
    # seq: [batch, seq_len]
    # 返回: [batch, 1, 1, seq_len]（适配 Attention 的 4D 输入）
    return (seq != pad_idx).unsqueeze(1).unsqueeze(2)

# 使用时：
# scores = scores.masked_fill(padding_mask == 0, float('-inf'))
```

### 5.4 常见误区

| 误区 | 正确理解 |
|:---|:---|
| "Mask 就是把值设为 0" | 错，是设为 $-\infty$，这样 softmax 后权重才是 0 |
| "Mask 只在推理时用" | 错，**训练时也必须用**，保证训练和推理一致 |
| "解码器的 Cross Attention 也需要 Mask" | 错，Cross Attention 看到源序列全部信息，不需要 Mask |

## 六、复杂度分析：O(L²) 到底意味着什么

### 6.1 Attention 的计算图

```
输入 X: [L, d]

Q = X @ W_Q:      [L, d] × [d, d] = O(L · d²)
K = X @ W_K:      O(L · d²)
V = X @ W_V:      O(L · d²)
scores = Q @ K^T: [L, d] × [d, L] = O(L² · d)
weights @ V:      [L, L] × [L, d] = O(L² · d)

总复杂度: O(L² · d + L · d²)
```

当 $L$（序列长度）很大时，**$L^2 \cdot d$ 是主要矛盾**。

### 6.2 复杂度对比

| 模型 | 序列长度 $L$ | Attention 复杂度 | 说明 |
|:---|:---|:---|:---|
| BERT | 512 | $O(L^2 \cdot d)$ | 标准 Attention |
| GPT-2 | 1024 | $O(L^2 \cdot d)$ | 上下文更长 |
| GPT-4 | ~32k+ | $O(L^2)$ | 用 Flash Attention 等优化 |
| Longformer | ~16k | $O(L)$ | 稀疏 Attention（滑动窗口 + 全局） |

### 6.3 L² 问题的实际影响

假设 $d = 512$，比较不同序列长度：

| $L$ | Attention 矩阵大小 | float32 大小 | 可行性 |
|:---|:---|:---|:---|
| 512 | $512^2 = 262,144$ | ~1 MB | ✅ 无压力 |
| 2048 | $2048^2 = 4,194,304$ | ~16 MB | ✅ 可接受 |
| 16384 | $16384^2 = 268M$ | ~1 GB | ⚠️ 显存瓶颈 |
| 100k | $10^{10}$ | ~40 GB | ❌ 无法直接处理 |

这就是为什么 LLM 有**上下文长度限制**的根本原因。

### 6.4 主流优化方向

**Flash Attention（2022）**：把 Attention 计算从显存瓶颈变成**计算瓶颈**，通过分块计算避免-materialize 整个 $L \times L$ 矩阵。速度提升 2-4×，显存减少 10-20×。

**Sparse Attention**：不是每个位置都看所有位置，而是：
- 滑动窗口：只看邻近 $w$ 个位置（复杂度 $O(L \cdot w)$）
- 全局位置：选几个特殊 token（如 [CLS]）与所有位置交互
- 随机注意力：随机选 $r$ 个位置交互

**Linear Attention**：把 $\exp(QK^T)$ 换成低秩分解，将复杂度降到 $O(L \cdot d^2)$ 或 $O(L \cdot d)$，代价是表达能力下降。

## 七、综合代码：完整 Transformer Attention Block

```python
class TransformerLayer(nn.Module):
    """
    一个完整的 Transformer 编码器层
    （解码器层多一个 Cross Attention，结构类似）
    """
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, num_heads)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),          # GPT 系列用 GELU，原始论文用 ReLU
            nn.Linear(d_ff, d_model),
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Self-Attention + 残差连接 + LayerNorm
        attn_out, _ = self.self_attn(x, x, x, mask)
        x = self.norm1(x + self.dropout(attn_out))

        # FFN + 残差连接 + LayerNorm
        ffn_out = self.ffn(x)
        x = self.norm2(x + self.dropout(ffn_out))

        return x
```

## 八、常见误区总结

| # | 误区 | 正确理解 |
|:---:|:---|:---|
| 1 | "$QK^T$ 不用缩放也能工作" | 小 $d_k$ 可以，$d_k \ge 512$ 时几乎无法收敛 |
| 2 | "Masked Attention 让模型变笨" | 不会，它只是强制模型自回归分解联合概率 |
| 3 | "多头就是把 Attention 多跑几遍取平均" | 是拼接后投影，不同头捕捉不同语义关系 |
| 4 | "Attention 矩阵就是相关性矩阵" | 是的，但学习出来的相关性，不是手动标注的 |
| 5 | "线性 Attention 比 softmax Attention 更好" | 在长序列上有优势，但表达能力通常更弱 |

## 九、核心知识点

1. **Q/K/V 三角**：Query=查询请求，Key=匹配钥匙，Value=实际内容，通过不同线性投影得到
2. **Scaled Dot-Product**：除以 $\sqrt{d_k}$ 让点积方差归一化，防止 softmax 饱和
3. **Multi-Head**：把 $d$ 维分成 $h$ 个头，每个头独立学一种语义关系模式，最后拼接投影
4. **Masked Attention**：解码器必须遮住未来位置，用 $-\infty$ + softmax 实现
5. **复杂度瓶颈**：$O(L^2 \cdot d)$ 是 LLM 上下文长度的根本限制，Flash Attention 是主流解法

## 十、思考题答案

**思考题 1**：均值 $= 0$，方差 $= 64$。（因为 $d_k = 64$，方差累加）

**思考题 2**：未缩放方差 $= 64$，缩放后方差 $= 1$。（$\sqrt{64} = 8$，$64 / 8^2 = 1$）

**思考题 3**：每个头 $d_k = d_{\text{model}} / h = 512 / 8 = 64$。参数量：$3 \times d_{\text{model}}^2 + d_{\text{model}}^2 = 4d_{\text{model}}^2 = 4 \times 512^2 \approx 1$ MB（仅 Attention 部分）

**思考题 4**：矩阵大小 $100 \times 100 = 10,000$。float32 每个元素 4 字节，显存约 $10,000 \times 4 = 40$ KB。**但实际上** $L = 100$ 很小，真正的瓶颈在 $L = 100k$ 级别。
