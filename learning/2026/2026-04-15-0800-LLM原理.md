## 📌 今日课题
**DPO：直接偏好优化/无需PPO/偏好数据集构建**

---

## 📖 完整讲解

## 原理

DPO（Direct Preference Optimization）= 直接用偏好数据优化模型，无需强化学习。

传统 RLHF 的问题：
- 三阶段流程复杂（RM训练 + PPO + 调参）
- PPO 超参众多，容易训崩
- 需要大量计算资源

DPO 的核心创新：把 RLHF 的 reward maximization 问题转化为「直接从偏好数据学习」。

### 直观理解

传统 RLHF：学一个 reward model -> 用 PPO 最大化 reward

DPO：直接学「什么是好答案 vs 坏答案」，不需要中间的 reward model

```python
# DPO Loss（简化）
# y_w = chosen (好答案), y_l = rejected (坏答案)
loss = -log(sigmoid(log_pi(y_w|x) - log_pi(y_l|x)))
```

目标是让模型给好答案的概率高于坏答案。

### DPO vs PPO

| | DPO | PPO |
|:---|:---|:---|
| 复杂性 | 低（一阶段）| 高（三阶段）|
| 稳定性 | 好 | 容易训崩 |
| 效果 | 略低于 PPO | 最好 |
| 适用场景 | 简单任务 | 复杂推理/代码 |

DPO 简单高效，但复杂任务（如复杂代码生成，数学推导）PPO 仍然领先。
