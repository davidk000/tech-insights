## 📖 完整讲解

## 原理

CoT（Chain-of-Thought）= 让模型「把思考过程说出来」，而不是直接给答案。

```
Q: 小明有3个苹果，小红给了他2个，小明又丢了1个，小明现在有几个？
A: 3 + 2 - 1 = 4，所以小明现在有4个苹果。
```

没有 CoT 的模型可能直接猜。有 CoT 的模型会一步步推理——准确率显著提升。

为什么有效？
- 强迫模型生成中间步骤 = 强迫模型进行更「长」的推理
- 中间步骤本身提供了更好的「prompt context」

### ReAct（Reasoning + Acting）

CoT 的问题：模型只能「想」，不能「做」（查资料、搜索等）。

ReAct 的思路：推理 + 行动交替进行。

```
问题：现在的美国总统是谁？
Thought: 我需要查一下最新信息。
Action: search[美国现任总统]
Observation: Joe Biden is the current US President.
Thought: 根据搜索结果，答案是 Joe Biden。
Final Answer: Joe Biden.
```

### Tree-of-Thought

更进一步：不只一条推理链，而是探索多条推理路径，选取最优。适合需要「搜索」或「规划」的问题（如数学证明、战略游戏）。
