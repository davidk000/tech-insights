## 一、背景：为什么 Transformer 需要位置编码

Transformer 的核心是 Self-Attention，它的核心运算是：

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

这个运算对输入序列的元素是**置换不变**（permutation-invariant）的——无论 token 出现在第1位还是第100位，如果 Q/K/V 相同，输出就相同。attention 机制本身不携带任何位置信息。

RNN/LSTM 的隐状态沿着时间步传递，天生带有顺序信息。CNN 通过卷积核的局部感受野隐含了相对位置。但 Transformer 必须**显式注入**位置信号，否则 "狗咬人" 和 "人咬狗" 在 attention 眼里毫无区别。

这就是位置编码（Positional Encoding, PE）要解决的问题。

## 二、Sinusoidal PE：Attention is All You Need 的原始方案

### 2.1 数学形式

原始 Transformer（Vaswani et al., 2017）提出的方案：

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

其中：
- `pos` 是 token 在序列中的位置（0-indexed）
- `i` 是维度索引（0 到 d_model/2）
- `d_model` 是 embedding 维度

每个位置的编码是一个 d_model 维向量，前半维度用 sin，后半维度用 cos。PE 直接加到输入 embedding 上（加性而非拼接）：

```
x' = x + PE(pos)
```

### 2.2 直观理解

Sinusoidal PE 的核心洞察是：**用不同频率的周期函数编码位置**。

- 低频维度（i 较小）：周期很长，覆盖很长距离的位置差异
- 高频维度（i 较大）：周期很短，能精细区分相邻位置

类比：钢琴键盘，低音弦振动慢（长周期），高音弦振动快（短周期）。每个音高（位置）都是所有频率的混合，这使得模型可以从不同"频率带"提取位置信息。

### 2.3 为什么有效：相对位置的线性表示

一个关键性质：Sinusoidal PE 可以表示**相对位置**，且是线性形式。

考虑 PE(pos+k) 和 PE(pos) 的内积——它们之间的差异仅依赖于相对偏移 k，而不依赖绝对位置。这意味着 attention 权重中自然携带相对位置信息。

Sin(ω·(pos+k)) = Sin(ω·pos)·Cos(ω·k) + Cos(ω·pos)·Sin(ω·k)

这是一个线性组合，其中 k（相对位置）可以作为系数被提取出来。

### 2.4 缺点

1. **外推能力差**：训练时最长序列是 512，测试时输入 1024？超出范围的 PE 值是全新的，模型没有见过
2. **不可学习**：参数是固定的（没有可训练的权重），只能靠模型自己适应
3. **绝对位置编码**：更适合需要绝对位置而非相对位置的任务

### 2.5 代码实现

```python
import numpy as np
import torch
import math

def sinusoidal_pe(max_len: int, d_model: int) -> torch.Tensor:
    """
    生成 Sinusoidal 位置编码
    max_len: 最大序列长度
    d_model: embedding 维度（必须为偶数）
    返回: (max_len, d_model) 的 PE 矩阵
    """
    assert d_model % 2 == 0, "d_model must be even"
    
    pe = torch.zeros(max_len, d_model)
    position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)  # (max_len, 1)
    
    # 计算除数项: 10000^(2i/d_model)
    div_term = torch.exp(
        torch.arange(0, d_model, 2, dtype=torch.float) *
        (-math.log(10000.0) / d_model)
    )  # (d_model/2,)
    
    pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维度
    pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维度
    
    return pe  # (max_len, d_model)


class SinusoidalPositionalEncoding(torch.nn.Module):
    def __init__(self, d_model: int, max_len: int = 5000):
        super().__init__()
        self.pe = sinusoidal_pe(max_len, d_model)  # 不是参数，不参与梯度更新
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch, seq_len, d_model)
        """
        seq_len = x.size(1)
        return x + self.pe[:seq_len]  # broadcast: (batch, seq_len, d_model)
```

## 三、RoPE 旋转位置编码：让 Query 和 Key 旋转

### 3.1 核心思想

RoPE（Rotary Position Embedding，苏剑林，2021）的核心洞察是：**不在 embedding 空间直接加位置信息，而是让 Q 和 K 在旋转后的空间中做 attention**。

形式上，RoPE 将 Q/K 向量在每两个相邻维度上做二维旋转：

```
q'_m = R(θ, m) · q_m
k'_n = R(θ, n) · k_n
```

旋转矩阵 R(θ, m) 是：

```
R(θ, m) = | cos(mθ)  -sin(mθ) |
          | sin(mθ)   cos(mθ) |
```

attention score 变成：

```
(q'_m)^T k'_n = q_m^T R(θ, m)^T R(θ, n) k_n
             = q_m^T R(θ, n-m) k_n
```

**关键性质**：最终的 attention score 只依赖于相对位置 n-m，**完全消除了对绝对位置的依赖**。

### 3.2 为什么叫"旋转"

在二维子空间里，乘以 R(θ, m) 就是把向量逆时针旋转 mθ 弧度。位置 m 越大，旋转角度越大。q_m 旋转 mθ，k_n 旋转 nθ，它们的内积只关心相对旋转角度差 (n-m)θ。

这类似于复数乘法：e^(imθ) · e^(inθ) = e^(i(m+n)θ)，但 RoPE 的设计使得结果是 e^(i(n-m)θ)，恰好编码相对位置。

### 3.3 多维推广：rope 中的频率设置

实际实现中，RoPE 对相邻维度对使用不同的旋转频率：

```
θ_i = 10000^(-2i/d)  (i = 0, 1, 2, ..., d/2-1)
```

这与 Sinusoidal PE 的分母项相同，只是应用方式从"加性偏置"变成了"旋转操作"。

### 3.4 RoPE vs Sinusoidal：核心区别

| 维度 | Sinusoidal | RoPE |
|:---|:---|:---|
| 注入方式 | 加到 embedding 上 | 旋转 Q/K 向量 |
| 相对位置 | 隐式（通过内积性质导出） | **显式**，attention 自然携带 |
| 绝对位置 | 是 | 否 |
| 可学习 | 否 | 否（但可结合 ALiBi 变成可学） |
| 外推 | 差（超出范围的值陌生） | 好（旋转是连续操作） |

### 3.5 代码实现（PyTorch）

```python
import torch
import math

def precompute_freqs_cis(dim: int, end: int, theta: float = 10000.0) -> torch.Tensor:
    """
    预计算旋转角度（复数形式 e^(i*θ)）
    dim: 每组旋转的维度（通常 dim=2）
    end: 最大位置
    theta: 基础频率参数
    返回: (end, dim//2) 的复数张量
    """
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2, dtype=torch.float) / dim))
    t = torch.arange(end, dtype=torch.float)
    freqs = torch.outer(t, freqs)  # (end, dim//2)
    freqs_cis = torch.polar(torch.ones_like(freqs), freqs)  # 复数: e^(i*freqs)
    return freqs_cis  # shape: (end, dim//2)


def apply_rotary_pos_emb(q: torch.Tensor, k: torch.Tensor, freqs_cis: torch.Tensor) -> tuple:
    """
    对 q 和 k 应用 RoPE
    q: (batch, seq_len, n_heads, head_dim)  — head_dim 必须是偶数
    k: 同上
    freqs_cis: (seq_len, head_dim//2) 复数张量
    """
    batch_size, seq_len, n_heads, head_dim = q.shape
    assert head_dim % 2 == 0
    
    # reshape: (batch, seq_len, n_heads, head_dim//2, 2) -> (batch*n_heads*seq_len, head_dim//2, 2)
    q_complex = torch.view_as_complex(q.float().reshape(*q.shape[:-1], -1, 2))
    k_complex = torch.view_as_complex(k.float().reshape(*k.shape[:-1], -1, 2))
    
    # freqs_cis: (seq_len, head_dim//2) -> broadcast 到 batch/n_heads 维度
    freqs_cis = freqs_cis[:seq_len]  # (seq_len, head_dim//2)
    
    # 复数乘法：旋转
    q_rotated = torch.view_as_real(q_complex * freqs_cis).flatten(-2)
    k_rotated = torch.view_as_real(k_complex * freqs_cis).flatten(-2)
    
    return q_rotated.view_as(q), k_rotated.view_as(k)


# 使用示例
batch, seq_len, n_heads, head_dim = 2, 128, 8, 64
q = torch.randn(batch, seq_len, n_heads, head_dim)
k = torch.randn(batch, seq_len, n_heads, head_dim)

freqs_cis = precompute_freqs_cis(head_dim, seq_len)
q_rot, k_rot = apply_rotary_pos_emb(q, k, freqs_cis)
```

### 3.6 外推性能：为什么 RoPE 外推更好

Sinusoidal 在位置 1024 时产生的 embedding 与训练时见过的任何位置都不同。RoPE 则是对 Q/K 做旋转，旋转角度 θ·pos 是连续函数——1024 只是比 512 旋转了更多角度，但操作方式完全一致，没有"陌生的值"。

实验数据（LLaMA 等模型）：
- 位置 2048 外推：RoPE 相比 Sinusoidal，PPL 下降幅度小得多
- 结合位置插值（PI）和微调，可以将上下文窗口扩展到 32k+

## 四、ALiBi：不需要位置编码的位置编码

### 4.1 核心思想

ALiBi（Attention with Linear Biases，Press et al., 2021）彻底放弃了"在输入上加位置信息"的思路，转而在 attention score 上加一个线性偏置：

```
score(q_i, k_j) = (q_i · k_j) - |i - j| · λ
```

其中 λ 是每层每头的固定标量（不同 head 可以不同），|i-j| 是相对距离。

没有加性的 PE，没有旋转，就是一个简单的与距离成正比的扣分项——距离越远的 token，attention score 越低。

### 4.2 为什么有效

1. ** inductive bias**：语言中相邻词之间的相关性通常更强，这是一个合理的先验
2. **无需学习**：λ 是手工设计的，不需要参数，避免了过拟合位置信息
3. **天然外推**：相对距离 |i-j| 可以超出训练时的最大距离，超出范围的惩罚仍然是线性的、合理的
4. **与 RoPE 兼容**：ALiBi 的偏置是加在 attention score 上的，与 RoPE 的 Q/K 旋转操作正交，可以叠加使用（Bloombergs AI 等）

### 4.3  λ 的设置

原论文建议每层每头可以有独立的 λ，但实践中常用简化方案：

```
λ_i = 1 / 2^(8i/h)  (i = 1, 2, ..., h)
```

其中 h 是总头数，i 是头索引的秩。这确保不同头有不同的距离惩罚力度。

### 4.4 代码实现

```python
import torch
import torch.nn.functional as F

def build_alibi_slopes(n_heads: int, key_chunk_size: int = 32) -> list:
    """
    生成 ALiBi 的 λ 斜率参数
    n_heads: 注意力头数量
    key_chunk_size: 块大小（用于 attention mask 的稀疏化，可选）
    返回: list of λ 值，每头一个
    """
    # 原始论文公式: λ_i = 1 / 2^(8i/h)
    # 归一化到 (-1/h, 1/h) 范围
    def get_slopes_power(n: int):
        # 计算 2^(-8/2^log_n) = 2^(-8/2^log_n) ... 简化版
        start = 2 ** (-(2 ** -(math.log2(n) - 3)))
        ratio = start
        return [start * ratio ** i for i in range(n)]
    
    # PyTorch 官方实现
    _ALIBI_POW = 2 ** (-(2 ** -(math.log2(n_heads) - 3)))
    slopes = torch.tensor(
        [1.0 * _ALIBI_POW ** (i + 1) for i in range(n_heads)]
    )
    # 归一化
    slopes = slopes / torch.max(slopes, dim=-1, keepdim=True)[0]
    return slopes.tolist()


def alibi_attention_mask(seq_len: int, n_heads: int, device: torch.device) -> torch.Tensor:
    """
    构建 ALiBi 的 attention mask（加性）
    返回: (n_heads, seq_len, seq_len) 的下三角 mask
    """
    slopes = torch.tensor(build_alibi_slopes(n_heads), device=device)  # (n_heads,)
    
    # 生成下三角矩阵（相对距离）
    # arange: [0, 1, 2, ..., seq_len-1]
    # 减去转置得到相对距离 (i - j)
    alibi = torch.arange(seq_len, device=device)  # (seq_len,)
    alibi = alibi[None, :] - alibi[:, None]       # (seq_len, seq_len), 下三角为负
    
    # 取绝对值，然后乘以 slopes[:, None, None] 应用到每头
    alibi = alibi.abs().unsqueeze(0) * slopes[:, None, None]  # (n_heads, seq_len, seq_len)
    
    # ALiBi 是减法偏置，所以 mask 是负值（在 attention 计算时加上）
    return alibi  # (n_heads, seq_len, seq_len)


# 使用示例
seq_len, n_heads = 128, 8
mask = alibi_attention_mask(seq_len, n_heads, device='cpu')
# mask[i, j, k] = -|j-k| * λ_i  (偏置项)
# 在 attention 中：score = qk^T + mask（而不是减去）
```

### 4.5 ALiBi vs RoPE 对比

| 维度 | RoPE | ALiBi |
|:---|:---|:---|
| 实现层次 | Q/K 向量变换 | Attention score 后处理 |
| 可学习 | 不可学习（固定旋转） | 不可学习（固定斜率） |
| 相对位置 | ✅ 隐式通过旋转性质 | ✅ **显式**直接编码 |
| 外推能力 | 强（旋转连续性） | 强（相对距离无上界） |
| 与 flash attention 兼容 | ✅（需修改 kernel） | ✅ |
| 计算开销 | 极小（旋转操作） | 极小（加一个偏置矩阵） |
| 典型应用 | LLaMA, PaLM, Qwen | Bloombergs AI, MPT |

## 五、位置编码的实践对比与选择

### 5.1 主流开源模型采用

| 模型 | 位置编码 | 外推方案 |
|:---|:---|:---|
| BERT | Sinusoidal（不可学） | 无（固定长度） |
| GPT-2 | Sinusoidal（不可学） | 无 |
| GPT-3 | Sinusoidal（不可学） | 无 |
| LLaMA 1/2 | RoPE | 位置插值（PI）+ 微调 |
| LLaMA 3 | RoPE + YARN | PI + 微调 |
| Mistral | RoPE | 位置插值 |
| Qwen | RoPE | PI + NTK-aware scaling |
| MPT | ALiBi | 原生支持长上下文 |
| Bloomberg AI | RoPE + ALiBi | 组合增强外推 |
| GLM | RoPE | 位置插值 |

### 5.2 实践选择建议

**选 RoPE 如果**：
- 需要较好的外推能力（超过训练长度）
- 希望 Q/K 之间的 attention 自然携带相对位置信息
- 主流开源生态（如 vLLM、TGI 都对 RoPE 有良好支持）

**选 ALiBi 如果**：
- 追求极简实现，不需要任何位置 embedding 参数
- 需要原生外推，不需要微调
- 结合 RoPE 使用进一步增强外推

**不要用 Sinusoidal PE 如果**：
- 需要超过训练长度的推理（外推问题会很明显）

## 六、位置插值（PI）与 NTK-Aware Scaling

当需要将模型上下文窗口从 L 扩展到 L'（L' > L）时，直接外推会导致分布偏移。两种常用技术：

### 6.1 位置插值（Positional Interpolation，PI）

将位置索引等比例缩放：pos' = pos × (L / L')，将新的长位置范围压缩到训练时的范围内。

```
PE_PI(pos') = PE_original(pos × L/L')
```

然后在压缩后的位置上做短时微调（如 1000 steps）。

效果：LLaMA 7B 从 2048 → 8192 上下文，PPL 基本保持不下降。

### 6.2 NTK-Aware Scaling（江湖昵称）

基于神经切线核（NTK）理论的分析，认为简单地等比例缩放不是最优的。NTK-aware 方法对 RoPE 的频率进行非线性缩放：

```
θ_i' = θ_i × α^(2i/d)  (仅对高频维度做更大缩放)
```

这样高频维度（精细的位置区分能力）被更激进地保留，而低频维度被压缩。

YaRN（Yet another RoPE extensioN）在此基础上进一步改进，结合了 PI 和 NTK 的思想。

### 6.3 外推 vs 插值的选择

- **微调成本允许**：PI + 短微调 是最稳妥的方案
- **零微调场景**：NTK-aware scaling 或 ALiBi
- **两者结合**：RoPE + NTK + 极短微调 = 目前最优（Bake-off MTG，2024）

## 七、常见误区

### 误区1：PE 可以直接拼接而非相加
原始 Transformer 使用加性，因为加性可以让 attention 机制自然地学习到位置偏差。如果拼接，维度翻倍，后面的投影矩阵 W^Q/W^K/W^V 需要额外参数来补偿。

### 误区2：RoPE 只能用于 2 维旋转
实际实现中，RoPE 在相邻维度对（2i, 2i+1）上做二维旋转。这是一个技巧：在更高维上做独立的二维旋转块，数学上等价于在对应二维子空间中的旋转。

### 误区3：ALiBi 不适合生成任务
恰恰相反，ALiBi 在 MPT 等模型上展示了优秀的生成质量和长距离依赖建模能力。ALiBi 的 attention score 自然衰减更适合自回归生成。

### 误区4：RoPE 外推不需要任何微调
对于较短的外推（如 10-20% 增长），零微调可能勉强可用。对于更大的扩展（如 4x），必须配合短微调，否则模型质量会明显下降。

### 误区5：位置编码只在 embedding 层应用一次
实际上，RoPE 在每层都应用——每一层的 Q 和 K 都要旋转。这确保了每一层的 attention 都携带正确的位置信息。

## 十、总结

```
位置编码演进路径：
Sinusoidal (2017) → RoPE (2021) → ALiBi (2022) → PI + NTK + YaRN (2023)
  加性偏置       旋转Q/K       线性衰减偏置    自适应外推

核心权衡：
- 绝对位置 vs 相对位置
- 可学习 vs 固定
- 外推能力 vs 训练复杂度
- 实现复杂度 vs 兼容性
```

**本节要点**：
- Attention 天然不携带位置信息，必须显式注入
- Sinusoidal PE 用不同频率的周期函数编码位置，可表示相对位置，但外推差
- RoPE 对 Q/K 做旋转，attention score 自然携带相对位置，外推能力好，是当前主流
- ALiBi 在 attention score 上加线性衰减偏置，零参数，原生外推，可与 RoPE 叠加
- 位置插值（PI）和 NTK-aware scaling 是处理超长上下文的标准工程手段
