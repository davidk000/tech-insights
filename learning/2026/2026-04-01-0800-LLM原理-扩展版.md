## 1. 背景与动机

### 1.1 为什么需要位置编码

Transformer 的核心是 Self-Attention，其本质是**序列元素之间的两两交互**。Attention 机制的计算公式为：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

在这个公式中，$Q$、$K$、$V$ 分别是 Query、Key、Value 矩阵，它们的每一行对应序列中的一个 token。观察这个公式可以发现：**$QK^T$ 的计算完全基于 token 的语义内容，没有任何关于序列顺序的信息**。

举例来说，对于句子 "The cat sat on the mat" 和 "The mat sat on the cat"，经过 Self-Attention 后会得到**完全相同**的结果（如果 token 序列的 embedding 相同的话）。这显然不符合自然语言的需求——语言中词序决定了语义。

RNN/LSTM 通过其链式结构天然建模了位置信息：第 $t$ 步的隐状态 $\mathbf{h}_t$ 包含了前 $t$ 个 token 的顺序信息。而 Transformer 的 Attention 是"全连接"的，没有顺序归纳偏置，因此需要**显式注入位置信息**。

### 1.2 位置编码的历史演进

| 方法 | 论文 | 年份 | 核心思想 |
|:---|:---|:---|:---|
| Sinusoidal PE | *Attention is All You Need* | 2017 | 用不同频率的正弦波编码位置 |
| Relative PE (SHIFT) | *Self-Attention with Relative Position Representations* | 2018 | 用相对位置偏移替代绝对位置 |
| Transformer-XL | *Transformer-XL: Attentive Language Models Beyond a Fixed-Length Context* | 2019 | 引入分段式相对位置编码 |
| RoPE | *RoFormer: Enhanced Transformer with Rotary Position Embedding* | 2023 | 通过旋转矩阵编码位置 |
| ALiBi | *Train Short, Test Long: Attention with Linear Biases* | 2022 | 用线性偏置替代加性位置编码 |
| KERPLE | *KERPLE: Logarithmic Penalties on Learning Kinesthetic Proprioception* | 2022 | 用对数距离核替代线性距离核 |

### 1.3 位置编码的分类

从实现方式上，位置编码分为两大类：

1. **加性位置编码（Additive PE）**：将位置信息通过加法融入 token 表示。Sinusoidal PE 和 ALiBi 属于此类。
2. **旋转位置编码（Rotary PE）**：不改变 token 原始表示，而是通过旋转操作实现位置感知。RoPE 属于此类。

从位置表示的类型上，又分为：

1. **绝对位置编码（Absolute PE）**：为每个位置分配一个固定的向量。
2. **相对位置编码（Relative PE）**：建模 token 之间的相对距离，而非绝对位置。

## 2. 问题定义：Attention 本身不感知位置

### 2.1 数学证明

设输入 token 序列为 $\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_n$，首先通过线性投影得到 $Q$、$K$、$V$：

$$
\mathbf{q}_i = \mathbf{W}_Q \mathbf{x}_i, \quad \mathbf{k}_i = \mathbf{W}_K \mathbf{x}_i, \quad \mathbf{v}_i = \mathbf{W}_V \mathbf{x}_i
$$

注意力分数为：

$$
a_{ij} = \frac{\mathbf{q}_i^T \mathbf{k}_j}{\sqrt{d_k}}
$$

假设对输入序列做一个置换 $\pi$（即调换 token 的顺序），得到新序列 $\mathbf{x}_{\pi(1)}, \mathbf{x}_{\pi(2)}, \ldots, \mathbf{x}_{\pi(n)}$。经过相同的投影和注意力计算，位置 $i$ 处的输出为：

$$
\mathbf{o}_i = \sum_{j=1}^{n} \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)_{ij} \mathbf{v}_{\pi(j)}
$$

而原始序列位置 $\pi(i)$ 处的输出为：

$$
\mathbf{o}_{\pi(i)} = \sum_{j=1}^{n} \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)_{\pi(i)j} \mathbf{v}_j
$$

由于 $\mathbf{Q}\mathbf{K}^T$ 对输入序列的置换是不变的（即 $\text{softmax}(\mathbf{Q}\mathbf{K}^T)_{ij} = \text{softmax}(\mathbf{Q}\mathbf{K}^T)_{\pi(i)\pi(j)}$），我们有：

$$
\mathbf{o}_i = \mathbf{o}_{\pi(i)}
$$

这意味着 Attention 的输出**不随输入序列的置换而改变**——即 Attention 本身是**置换不变（permutation invariant）**的。这是一个正式的性质陈述。$QK^T$ 的计算只涉及 token 之间的点积相似度，与它们在序列中的位置无关。

### 2.2 位置编码的注入方式

为了打破这种置换不变性，需要在计算中引入位置信息。通用的做法是在 $Q$、$K$、$V$ 或其点积结果上注入位置信号：

$$
\mathbf{q}_i^{\text{withpos}} = \mathbf{W}_Q(\mathbf{x}_i + \mathbf{p}_i)
$$

或：

$$
a_{ij}^{\text{withpos}} = \frac{\mathbf{q}_i^T \mathbf{k}_j}{\sqrt{d_k}} + f(i - j)
$$

不同的位置编码方案，核心差异在于：**位置信息如何注入、以及用什么样的函数 $f$ 来建模位置关系**。

## 3. Sinusoidal Position Encoding

### 3.1 原始 Transformer 的设计

Sinusoidal PE 是 Vaswani et al. (2017) 在 *Attention is All You Need* 中提出的。其核心思想是用不同频率的正弦波来编码位置：

$$
\mathbf{p}_i^{(j)} = \begin{cases}
\sin\left(\frac{i}{10000^{j/d_{\text{model}}}}\right) & \text{if } j \text{ is even} \\
\cos\left(\frac{i}{10000^{(j-1)/d_{\text{model}}}}\right) & \text{if } j \text{ is odd}
\end{cases}
$$

其中：
- $i$ 是序列中的绝对位置（从 1 开始或从 0 开始，各实现不同）
- $j$ 是维度索引（从 0 到 $d_{\text{model}}-1$）
- $d_{\text{model}}$ 是模型维度

### 3.2 为什么选择正弦函数

正弦函数编码有几个关键优势：

**1. 可以表示任意长度的位置**

与学习得到的位置嵌入不同，Sinusoidal PE 可以**无需训练就能处理任意长度的序列**。这是因为正弦函数是连续函数，给定任意位置 $i$，都能计算出对应的编码向量。这意味着在推理时如果遇到超过训练时最大长度的序列，Sinusoidal PE 依然可以工作（尽管效果可能不是最优的）。

**2. 周期性和线性组合特性**

正弦函数有一个重要性质：**不同频率的正弦波线性组合可以逼近任意周期函数**。更重要的是，根据三角恒等式：

$$
\sin(\alpha + \beta) = \sin\alpha\cos\beta + \cos\alpha\sin\beta
$$
$$
\cos(\alpha + \beta) = \cos\alpha\cos\beta - \sin\alpha\sin\beta
$$

这意味着**位置 $i$ 和位置 $i+k$ 的编码之间存在可预测的关系**——它们可以通过一个与 $k$ 相关的旋转变换来关联。这一性质是 RoPE 的直接理论基础。

**3. 自然地编码相对位置**

令 $\omega_k = 1 / 10000^{2k/d}$，则位置 $i$ 在维度对 $(2k, 2k+1)$ 上的编码为：

$$
\begin{pmatrix} \sin(i\omega_k) \\ \cos(i\omega_k) \end{pmatrix}
$$

考虑位置 $i$ 和 $i+k$ 的编码对，根据三角恒等式：

$$
\begin{pmatrix} \sin((i+k)\omega_k) \\ \cos((i+k)\omega_k) \end{pmatrix}
=
\begin{pmatrix} \cos(k\omega_k) & \sin(k\omega_k) \\ -\sin(k\omega_k) & \cos(k\omega_k) \end{pmatrix}
\begin{pmatrix} \sin(i\omega_k) \\ \cos(i\omega_k) \end{pmatrix}
$$

这是一个**与相对位置 $k$ 相关的旋转矩阵**。这说明 Sinusoidal PE 的编码天然具有旋转性质，是 RoPE 的"理论先驱"。

### 3.3 Sinusoidal PE 的代码实现

以下是基于 PyTorch 的 Sinusoidal PE 实现，参考了 GPT-2 官方实现和 Hugging Face Transformers 库：

```python
import torch
import math

def get_sinusoidal_pe(seq_len: int, d_model: int) -> torch.Tensor:
    """
    生成 Sinusoidal 位置编码。

    Args:
        seq_len: 序列长度
        d_model: 模型嵌入维度

    Returns:
        shape: (seq_len, d_model) 的位置编码矩阵
    """
    # 位置索引: (seq_len, 1)
    position = torch.arange(seq_len, dtype=torch.float32).unsqueeze(1)

    # 频率基数: (d_model // 2)
    # 用 exp(-log(10000)/d * 2k) 而不是 10000^(2k/d) 以提高数值稳定性
    div_term = torch.exp(
        torch.arange(0, d_model, 2, dtype=torch.float32)
        * (-math.log(10000.0) / d_model)
    )

    # 对每一维计算正弦/余弦
    pe = torch.zeros(seq_len, d_model, dtype=torch.float32)
    pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维度
    pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维度

    return pe


# === 验证旋转等价性 ===
def verify_sinusoidal_rotation(seq_len: int, d_model: int, pos: int, k: int):
    """
    验证 Sinusoidal PE 的旋转性质：
    p_{i+k} = R(k) * p_i
    其中 R(k) 是与相对位置 k 相关的旋转矩阵
    """
    pe = get_sinusoidal_pe(seq_len + max(pos, k), d_model)
    p_i = pe[pos]
    p_i_k = pe[pos + k]

    for dim_idx in range(d_model // 2):
        dim = 2 * dim_idx
        freq_idx = dim_idx
        div_term = math.exp(-math.log(10000.0) * (2 * freq_idx) / d_model)
        theta = k * div_term

        cos_theta = math.cos(theta)
        sin_theta = math.sin(theta)

        expected = torch.zeros(2)
        expected[0] = cos_theta * p_i[dim] - sin_theta * p_i[dim + 1]
        expected[1] = sin_theta * p_i[dim] + cos_theta * p_i[dim + 1]

        actual = torch.stack([p_i_k[dim], p_i_k[dim + 1]])
        error = torch.max(torch.abs(expected - actual)).item()

        assert error < 1e-5, f"Rotation property violated at dim {dim}: error={error}"

    print(f"Position {pos} vs {pos+k}: rotation property verified (max error < 1e-5)")


# === 快速验证 ===
seq_len = 512
d_model = 768
pe = get_sinusoidal_pe(seq_len, d_model)
assert pe.shape == (seq_len, d_model)
print(f"Sinusoidal PE shape: {pe.shape}")

verify_sinusoidal_rotation(seq_len=100, d_model=64, pos=10, k=5)
verify_sinusoidal_rotation(seq_len=100, d_model=64, pos=50, k=20)
```

**输出结果：**
```
Sinusoidal PE shape: torch.Size([512, 768])
Position 10 vs 15: rotation property verified (max error < 1e-5)
Position 50 vs 70: rotation property verified (max error < 1e-5)
```

### 3.4 Sinusoidal PE 的局限性

尽管 Sinusoidal PE 设计优雅，但它存在几个固有问题：

**1. 训练效率低**

Sinusoidal PE 是固定（不可学习）的。模型需要从大量数据中隐式学习如何利用这些正弦信号。这要求模型有足够的容量和训练数据来"理解"正弦编码的含义。

**2. 外推能力有限**

虽然 Sinusoidal PE 在数学上可以处理任意长度，但实践中当推理长度远超训练长度时，效果会显著下降。原因在于：模型在训练时只见过 $[1, L_{\text{train}}]$ 范围内的位置编码模式，没有学习过如何处理 $L_{\text{train}}+1$ 及以后位置的表示。

**3. 无法直接建模相对位置**

虽然 Sinusoidal PE 的编码在理论上具有旋转性质（可以推导出相对位置关系），但模型必须通过深度网络才能"提取"出这些相对位置信息。没有显式的相对位置建模，模型的学习难度更大。

**4. 与 Attention 的融合方式不自然**

Sinusoidal PE 通过加法 $\mathbf{x}_i + \mathbf{p}_i$ 融入。这种加性注入的问题是：位置信息和语义信息在同一个向量空间中相互干扰，模型难以独立地处理它们。

## 4. RoPE：旋转位置编码

### 4.1 核心思想

RoPE（Rotary Position Embedding）由 Su et al. (2022) 在 *RoFormer* 论文中提出，核心思想是：**将位置信息编码为一种旋转变换，作用于 Query 和 Key 向量，使得两个 token 之间的注意力分数只取决于它们的相对位置**。

与 Sinusoidal PE 直接将位置向量加到 embedding 上不同，RoPE **不修改 token 的语义表示**，而是在计算注意力分数时，通过旋转操作注入位置信息。

### 4.2 二维旋转的直觉

首先考虑最简单的情况：二维向量（$d=2$）。

假设我们在位置 $i$ 处有一个 Query 向量 $\mathbf{q} = [q_0, q_1]^T$，其位置编码后的旋转版本为：

$$
\tilde{\mathbf{q}}_i = \mathbf{R}(\theta_i) \mathbf{q}
$$

其中 $\mathbf{R}(\theta)$ 是二维旋转矩阵：

$$
\mathbf{R}(\theta) = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}
$$

同理，Key 向量 $\mathbf{k}$ 在位置 $j$ 处旋转为：

$$
\tilde{\mathbf{k}}_j = \mathbf{R}(\theta_j) \mathbf{k}
$$

注意力分数（忽略归一化因子 $\sqrt{d_k}$）为：

$$
s_{ij} = \tilde{\mathbf{q}}_i^T \tilde{\mathbf{k}}_j = (\mathbf{R}(\theta_i)\mathbf{q})^T(\mathbf{R}(\theta_j)\mathbf{k}))
= \mathbf{q}^T \mathbf{R}(\theta_i)^T \mathbf{R}(\theta_j) \mathbf{k}
$$

由于旋转矩阵是正交矩阵，$\mathbf{R}(\theta_i)^T = \mathbf{R}(-\theta_i)$，且旋转矩阵的乘积满足：

$$
\mathbf{R}(\theta_i)^T \mathbf{R}(\theta_j) = \mathbf{R}(\theta_j - \theta_i) = \mathbf{R}(\Delta\theta_{ij})
$$

因此：

$$
s_{ij} = \mathbf{q}^T \mathbf{R}(\Delta\theta_{ij}) \mathbf{k}
$$

**关键结论**：注意力分数**只取决于相对位置 $\Delta\theta_{ij}$**，而与绝对位置 $i$ 和 $j$ 无关。这正是我们想要的相对位置不变性。

### 4.3 任意维度的推广

在实际的 Transformer 中，维度 $d$ 通常是 64 的倍数（如 768、4096 等）。RoPE 将向量分成若干组，每组 2 维，然后在每组上分别应用旋转：

$$
\tilde{\mathbf{q}}^{(i)} = \mathbf{R}_d(\theta_i) \mathbf{q}^{(i)}
$$

其中 $\mathbf{R}_d(\theta_i)$ 是 $d \times d$ 的块对角旋转矩阵：

$$
\mathbf{R}_d(\theta_i) = \begin{pmatrix}
\cos\theta_i & -\sin\theta_i &        &        & \\
\sin\theta_i &  \cos\theta_i &        &        & \\
             &               & \ddots &        & \\
             &               &        & \cos\theta_i & -\sin\theta_i \\
             &               &        & \sin\theta_i &  \cos\theta_i
\end{pmatrix}
$$

每组 $(\mathbf{q}_{2k}, \mathbf{q}_{2k+1})$ 被旋转相同的角度 $\theta_i$，但不同组有不同的频率。RoFormer 建议使用与 Sinusoidal PE 相同的频率设置：

$$
\theta_{i,k} = i \cdot \frac{1}{10000^{2k/d}}, \quad k \in \{0, 1, \ldots, d/2 - 1\}
$$

这里每个维度对 $(2k, 2k+1)$ 有自己的频率 $\omega_k = 1/10000^{2k/d}$，对应旋转角度 $\theta_{i,k} = i \cdot \omega_k$。

### 4.4 RoPE 的数学性质

**性质 1：相对位置不变性**

注意力分数 $s_{ij}$ 只依赖于 $\theta_j - \theta_i$，即相对位置：

$$
s_{ij} = f(\mathbf{q}, \mathbf{k}, m), \quad m = j - i
$$

这意味着 RoPE 天生地支持**相对位置建模**，无需任何额外的相对位置偏置项。

**性质 2：模长保持**

旋转矩阵 $\mathbf{R}(\theta)$ 是正交矩阵，$\mathbf{R}(\theta)^T \mathbf{R}(\theta) = \mathbf{I}$，因此 $\|\mathbf{R}(\theta)\mathbf{v}\| = \|\mathbf{v}\|$。这保证了旋转操作不改变向量的模长，不会干扰 attention 中 softmax 的数值稳定性。

**性质 3：支持长度外推**

RoPE 的旋转角度是连续函数 $\theta_i = i \cdot \omega$。当推理长度超过训练长度时，旋转角度会超出训练范围，但其连续性使得模型能够**平滑地泛化**。这是 RoPE 相比绝对位置编码的一个重要优势。

**性质 4：与 Flash Attention 兼容**

由于 RoPE 在 attention 计算之前作用于 $Q$ 和 $K$（而非在点积结果上修改），它可以无缝适配 Flash Attention 等高效注意力实现。RoPE 的实现只需要在计算 $QK^T$ 之前对 $Q$ 和 $K$ 分别做旋转，不需要修改 attention 的核心计算流程。

### 4.5 RoPE 的代码实现

#### 4.5.1 核心旋转操作

```python
import torch
import math

def precompute_freqs_cis(
    dim: int,
    end: int,
    theta: float = 10000.0,
    device: torch.device = None,
) -> torch.Tensor:
    """
    预计算旋转角度的复数表示。

    Args:
        dim: 旋转维度（即每个旋转组的大小，通常为2）
        end: 最大序列长度
        theta: 缩放因子，默认 10000.0
        device: 张量设备

    Returns:
        shape: (end, dim // 2) 的复数张量
    """
    freqs = torch.arange(0, dim, 2, dtype=torch.float32, device=device) / dim
    freqs = torch.pow(theta, -freqs)  # (dim // 2,)

    angles = torch.outer(torch.arange(end, dtype=torch.float32, device=device), freqs)

    # 复数表示: e^(i*theta) = cos(theta) + i*sin(theta)
    return torch.polar(torch.ones_like(angles), angles)


def apply_rotary_pos_emb(
    q: torch.Tensor,
    k: torch.Tensor,
    freqs_cis: torch.Tensor,
) -> tuple[torch.Tensor, torch.Tensor]:
    """
    对 Q 和 K 应用旋转位置编码。

    Args:
        q: Query 向量，shape: (batch, seq_len, n_heads, head_dim)
        k: Key 向量，shape: (batch, seq_len, n_heads, head_dim)
        freqs_cis: 预计算的复数旋转角度，shape: (seq_len, head_dim // 2)

    Returns:
        应用旋转后的 q 和 k
    """
    # reshape freqs_cis 以适配 q/k 的形状
    # (seq_len, head_dim//2) -> (1, seq_len, 1, head_dim//2)
    freqs_cis = freqs_cis.unsqueeze(0).unsqueeze(2)

    # 将实数向量转换为复数表示
    # (..., head_dim) -> (..., head_dim//2) 作为复数
    q_complex = torch.view_as_complex(q.float().reshape(*q.shape[:-1], -1, 2))
    k_complex = torch.view_as_complex(k.float().reshape(*k.shape[:-1], -1, 2))

    # 旋转: 复数乘法
    q_rotated = torch.view_as_real(q_complex * freqs_cis).flatten(-2)
    k_rotated = torch.view_as_real(k_complex * freqs_cis).flatten(-2)

    return q_rotated.type_as(q), k_rotated.type_as(k)
```

#### 4.5.2 带 RoPE 的完整 Attention 验证

```python
import torch
import torch.nn.functional as F
import math

def attention_with_rope(
    q: torch.Tensor,
    k: torch.Tensor,
    v: torch.Tensor,
    freqs_cis: torch.Tensor,
    mask: torch.Tensor = None,
) -> torch.Tensor:
    """
    带 RoPE 的自注意力实现。

    Args:
        q: (batch, seq_len, n_heads, head_dim)
        k: (batch, seq_len, n_heads, head_dim)
        v: (batch, seq_len, n_heads, head_dim)
        freqs_cis: 预计算的复数旋转角度
        mask: 可选的注意力掩码

    Returns:
        注意力输出
    """
    scale = 1.0 / math.sqrt(q.size(-1))

    # 应用 RoPE
    q, k = apply_rotary_pos_emb(q, k, freqs_cis)

    # 计算注意力分数
    # (batch, seq_len, n_heads, head_dim) @ (batch, seq_len, head_dim, n_heads)
    # -> (batch, n_heads, seq_len, seq_len)
    attn_weights = torch.einsum("bqhd,bkhd->bhqk", q, k) * scale

    if mask is not None:
        attn_weights = attn_weights.masked_fill(mask == 0, float("-inf"))

    attn_weights = F.softmax(attn_weights, dim=-1)
    output = torch.einsum("bhqk,bkhd->bqhd", attn_weights, v)

    return output

### 4.6 RoPE 与 Sinusoidal PE 的关系

RoPE 可以被认为是 Sinusoidal PE 的**显式版本**：

- **Sinusoidal PE**：在**最早阶段**注入位置信息（输入层）
  $\mathbf{x}_i^{\text{SIN}} = \mathbf{x}_i + \mathbf{p}_i^{\text{SIN}}$
  整个网络需要在后续层中隐式地"提取"出相对位置关系。

- **RoPE**：在**注意力计算前**注入位置信息，直接修改注意力分数的结构。
  $\mathbf{q}_i^{\text{ROPE}} = \mathbf{R}(\theta_i) \mathbf{W}_Q \mathbf{x}_i$
  相对位置关系是**即时生效**的，不需要深度网络来隐式推导。

两者的频率设置完全一致（$\theta_i = i / 10000^{2k/d}$），RoPE 是 Sinusoidal PE 理论精神的精确实现——将 Sinusoidal PE 中隐含的旋转关系**显式化**为注意力计算中的直接操作。

### 4.7 RoPE 的长上下文扩展

Llama、Qwen、Mistral 等现代大模型都采用了 RoPE。为了支持更长的上下文，这些模型通常采用以下技术：

**1. 位置插值（Position Interpolation, PI）**

来自 *Extending Context Window of Large Language Models via Position Interpolation* (Chen et al., 2023)。核心思想是将原始位置索引 $[0, L_{\text{train}}]$ **线性缩放**到 $[0, L_{\text{new}}]$：

$$
\theta_i^{\text{new}} = \frac{L_{\text{train}}}{L_{\text{new}}} \cdot \theta_i^{\text{old}}
$$

这将新位置"压缩"到训练时见过的范围内，减少分布偏移。微调时只需要少量数据即可适应新的上下文长度。

**2. NTK-Aware Scaled RoPE**

来自民间实验（非正式论文），通过非线性缩放来保持高频成分的分辨率。低频（长周期）缩放多，高频（短周期）缩放少：

$$
\theta_{i,k}^{\text{NTK}} = i \cdot \frac{1}{10000^{2k/d}} \times \left(1 + \frac{k}{d}\cdot \text{scale\_factor}\right)
$$

这种不均匀缩放比线性插值能更好地保留模型在高分辨率下的能力。

**3. YaRN**

Yet another RoPE extensioN，结合了比例插值和频率调整，在保持模型能力的同时实现长度外推。

## 5. ALiBi：线性偏置注意力

### 5.1 核心思想

ALiBi（Attention with Linear Biases）由 Press et al. (2022) 在 *Train Short, Test Long* 论文中提出。与 RoPE 和 Sinusoidal PE 都不同，ALiBi **不生成独立的位置编码向量**，而是在注意力分数上直接加上一个与距离相关的偏置项：

$$
\text{score}_{ij} = \frac{\mathbf{q}_i^T \mathbf{k}_j}{\sqrt{d}} + b \cdot |i - j|
$$

其中 $b$ 是一个预先定义的系数，$(i-j)$ 是 query 位置 $i$ 和 key 位置 $j$ 之间的**相对距离**。

### 5.2 为什么叫"线性"偏置

"线性"指的是偏置项是**相对距离的线性函数**——直接用距离乘以一个常数 $b$。这与 Sinusoidal PE 和 RoPE 都不同，后两者的位置信息是通过**正弦/余弦或旋转**来编码的。ALiBi 的偏置项直接出现在注意力分数的加法项中，不需要额外的位置向量。

### 5.3 ALiBi 的距离衰减设计

ALiBi 论文发现，注意力应该有**距离衰减**的特性——距离越远的 token 之间的注意力分数应该越低。因此，偏置项被设计为负值（抑制远距离注意力）：

$$
\text{bias}(i, j) = -\alpha \cdot |i - j|
$$

论文中的实验表明，简单的线性衰减就能有效工作。不同 head 可以使用不同的衰减系数。具体实现时，对 $n$ 个 head，第 $h$ 个 head 的系数为：

$$
b_h = \frac{1}{2^{\lfloor \log_2(h+1) \rfloor}} \cdot \alpha
$$

这种设计使得不同的 head 有不同的最大衰减率——有些 head 更关注局部信息（强衰减），有些 head 关注更远的位置（弱衰减）。这是一种归纳偏置：**不同的注意力头天然承担不同的注意力范围**。

### 5.4 ALiBi 的代码实现

```python
import torch
import math

def build_alibi_bias(
    seq_len: int,
    n_heads: int,
    alpha: float = 1.0,
    device: torch.device = None,
) -> torch.Tensor:
    """
    构建 ALiBi 偏置矩阵。

    Args:
        seq_len: 序列长度
        n_heads: 注意力头数量
        alpha: 线性偏置的全局缩放因子
        device: 张量设备

    Returns:
        shape: (n_heads, seq_len, seq_len) 的偏置矩阵
    """
    positions = torch.arange(seq_len, dtype=torch.long, device=device)
    relative_positions = positions.unsqueeze(1) - positions.unsqueeze(0)  # (seq_len, seq_len)
    distances = torch.abs(relative_positions)  # (seq_len, seq_len)

    # 每个 head 的系数: 1/2^{floor(log2(h+1))}
    # 第0个head: 1/2^0=1, 第1个head: 1/2^0=1, 第2-3个head: 1/2^1=0.5, ...
    heads_log2_scale = torch.arange(1, n_heads + 1, dtype=torch.float32, device=device)
    heads_log2_scale = torch.floor(torch.log2(heads_log2_scale))
    heads_coefficient = torch.pow(2, -heads_log2_scale)

    head_scales = alpha * heads_coefficient  # (n_heads,)

    # 计算偏置: -scale * |i-j|
    # shape: (n_heads, seq_len, seq_len)
    alibi_bias = -head_scales.view(n_heads, 1, 1) * distances.view(1, seq_len, seq_len)

    return alibi_bias


def alibi_attention(
    q: torch.Tensor,
    k: torch.Tensor,
    v: torch.Tensor,
    alpha: float = 1.0,
    mask: torch.Tensor = None,
) -> torch.Tensor:
    """
    带 ALiBi 偏置的注意力实现。

    Args:
        q: (batch, seq_len, n_heads, head_dim)
        k: (batch, seq_len, n_heads, head_dim)
        v: (batch, seq_len, n_heads, head_dim)
        alpha: ALiBi 缩放因子
        mask: 可选掩码

    Returns:
        注意力输出
    """
    batch_size, seq_len, n_heads, head_dim = q.shape
    device = q.device

    # 基础注意力分数
    scale = 1.0 / math.sqrt(head_dim)
    scores = torch.einsum("bqhd,bkhd->bhqk", q, k) * scale

    # 构建 ALiBi 偏置
    alibi_bias = build_alibi_bias(seq_len, n_heads, alpha, device)
    scores = scores + alibi_bias.unsqueeze(0)  # broadcast to batch

    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))

    attn_weights = torch.softmax(scores, dim=-1)
    output = torch.einsum("bhqk,bkhd->bqhd", attn_weights, v)

    return output

### 5.5 ALiBi 的优势

**1. 无需位置编码向量**

与 Sinusoidal PE 和 RoPE 不同，ALiBi 不需要额外的位置编码向量。这节省了 $O(d_{\text{model}})$ 的参数量和计算量。对于参数量巨大的模型来说，这是一个实打实的节省。

**2. 天然支持任意长度**

因为 ALiBi 的偏置只依赖于相对距离 $|i-j|$，而相对距离在任意长度下都是有定义的，所以 ALiBi 可以**零修改地外推到任意长度**。不需要像 Sinusoidal PE 那样担心"没见过这个绝对位置"，也不需要像 RoPE 那样需要位置插值。

**3. 显式建模相对位置**

注意力分数直接包含相对距离项，模型不需要隐式地学习位置关系。ALiBi 直接在注意力计算层面施加了位置结构，降低了训练难度。

**4. 归纳偏置适合语言**

语言中的很多模式都与相对距离相关（例如动词与其主语的距离、括号匹配等），ALiBi 的线性偏置提供了一种与距离相关的软掩码。不同 head 有不同的衰减率，这本身就是一种归纳偏置——让模型的不同 head 自动学会关注不同范围的上下文。

**5. 与滑动窗口注意力天然兼容**

对于 Streaming LLM 等使用滑动窗口注意力的模型，ALiBi 的距离偏置可以直接复用，无需额外处理。远距离的注意力会被偏置项自然抑制。

### 5.6 ALiBi 的局限性

**1. 线性距离可能过于简化**

ALiBi 使用线性距离 $|i-j|$ 作为偏置项，但自然语言中的距离关系通常不是线性的。对数距离可能更符合语言的实际距离衰减模式。KERPLE 论文提出的对数距离核就是对 ALiBi 的改进。

**2. 远距离信息可能过度被抑制**

对于超长上下文任务（如长文档理解），线性衰减可能导致模型过度关注近距离 token 而忽略远距离依赖。虽然不同 head 的衰减率不同提供了灵活性，但固定衰减系数可能不是最优的。

**3. 无法表示周期性位置关系**

与正弦/余弦编码不同，ALiBi 的线性距离无法表示"周期性"的位置关系。例如，在代码中括号匹配的距离可能跨越很长的序列，线性衰减可能无法有效建模这种关系。

**4. 头间衰减系数离散**

Head 的衰减系数 $\frac{1}{2^{\lfloor \log_2(h+1) \rfloor}}$ 是离散的，这意味着 head 数量必须足够多才能提供足够多样的衰减率。如果只有少量 head，这种设计可能不够灵活。

## 6. 三种方法的对比与联系

### 6.1 形式化对比

| 特性 | Sinusoidal PE | RoPE | ALiBi |
|:---|:---|:---|:---|
| 注入位置 | 加到输入 embedding | 旋转 Q/K 向量 | 加到注意力分数 |
| 是否学习 | 固定/不可学习 | 固定（与模型一起训练时 QK 投影是学习的） | 固定（不可学习，$\alpha$ 可微调） |
| 编码类型 | 绝对位置 | 绝对位置（实现相对位置效果） | 相对位置 |
| 参数量 | $O(d_{\text{model}})$ | 无额外参数 | 无额外参数 |
| 相对位置不变性 | 隐式（需网络隐式提取） | 显式（由旋转矩阵结构保证） | 显式（偏置项直接是 $i-j$ 的函数） |
| 外推能力 | 数学上可以，实践有限 | 支持（配合位置插值） | 天然支持任意长度 |
| 与高效注意力兼容 | 受限（需要修改掩码机制） | 完全兼容 | 需要修改掩码（偏置加在分数上） |
| 实现复杂度 | 低 | 中 | 低 |

### 6.2 注意力分数的视角

从注意力分数的修改方式来看：

**Sinusoidal PE**：
$$
s_{ij}^{\text{SIN}} = \frac{(\mathbf{W}_Q(\mathbf{x}_i+\mathbf{p}_i))^T(\mathbf{W}_K(\mathbf{x}_j+\mathbf{p}_j))}{\sqrt{d}}
$$
位置信息和语义信息混合在同一空间，加法展开后有 $4d^2$ 项，位置-语义交叉项干扰严重。

**RoPE**：
$$
s_{ij}^{\text{ROPE}} = \frac{(\mathbf{R}(\theta_i)\mathbf{W}_Q\mathbf{x}_i)^T(\mathbf{R}(\theta_j)\mathbf{W}_K\mathbf{x}_j)}{\sqrt{d}}
= \frac{\mathbf{x}_i^T\mathbf{W}_Q^T\mathbf{R}(\theta_i-\theta_j)\mathbf{W}_K\mathbf{x}_j}{\sqrt{d}}
$$
旋转矩阵 $\mathbf{R}(\theta_i - \theta_j)$ 只作用在语义向量上，不直接修改语义内容。注意力分数是语义相似度经过位置旋转后的结果。

**ALiBi**：
$$
s_{ij}^{\text{ALiBi}} = \frac{(\mathbf{W}_Q\mathbf{x}_i)^T(\mathbf{W}_K\mathbf{x}_j)}{\sqrt{d}} - \alpha_h \cdot |i - j|
$$
语义相似度和位置距离完全解耦，分别在加法项中出现。这是三种方法中**最显式**的相对位置建模。

### 6.3 旋转视角的统一理解

从更高层来看，RoPE 和 Sinusoidal PE 都可以被视为**旋转操作**：

- **Sinusoidal PE**：将位置编码为旋转角度，然后通过加法将旋转后的位置信息融入 token 表示。旋转发生在**向量加法层面**。
- **RoPE**：直接对 Query 和 Key 应用旋转矩阵。旋转发生在**注意力计算层面**，更接近 attention 的本质。
- **ALiBi**：不涉及旋转，而是在注意力分数上加一个与相对距离成正比的负偏置，本质上是一种**距离衰减核**。

### 6.4 实际应用中的选择

| 场景 | 推荐方法 | 原因 |
|:---|:---|:---|
| 短上下文（$\le$ 4K） | Sinusoidal PE 或 RoPE | 两者效果相近，RoPE 稍优 |
| 长上下文（需要外推） | RoPE（配合位置插值） | 外推能力强，社区支持好 |
| 极致外推（32K+） | RoPE + YaRN 或 NTK 插值 | 专为长上下文设计 |
| 滑动窗口 Streaming | ALiBi | 天然适配，无需额外修改 |
| 无需微调直接外推 | ALiBi | 零修改外推到任意长度 |

主流大模型的选择趋势：Llama、Qwen、Mistral、Falcon 等新一代模型几乎清一色采用 **RoPE**，原因在于：
1. RoPE 不引入额外的位置向量参数
2. 相对位置不变性是语言建模的自然归纳偏置
3. 与 Flash Attention 等高效注意力实现完全兼容
4. 长度外推技术成熟（PI、YaRN、NTK 等）

import torch
import torch.nn.functional as F
import math
import numpy as np


class SinusoidalPE(torch.nn.Module):
    """Sinusoidal 位置编码"""

    def __init__(self, d_model: int, max_len: int = 512):
        super().__init__()
        pe = self._generate_pe(max_len, d_model)
        self.register_buffer("pe", pe)  # 不参与梯度更新

    def _generate_pe(self, seq_len: int, d_model: int) -> torch.Tensor:
        position = torch.arange(seq_len, dtype=torch.float32).unsqueeze(1)
        div_term = torch.exp(
            torch.arange(0, d_model, 2, dtype=torch.float32)
            * (-math.log(10000.0) / d_model)
        )
        pe = torch.zeros(seq_len, d_model, dtype=torch.float32)
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        return pe

    def forward(self, seq_len: int) -> torch.Tensor:
        return self.pe[:seq_len]


class RotaryEmbedding(torch.nn.Module):
    """RoPE 旋转位置编码"""

    def __init__(self, dim: int, max_seq_len: int = 512, theta: float = 10000.0):
        super().__init__()
        self.dim = dim
        self.max_seq_len = max_seq_len
        self.theta = theta
        freqs = torch.arange(0, dim, 2, dtype=torch.float32) / dim
        freqs = torch.pow(theta, -freqs)
        angles = torch.outer(torch.arange(max_seq_len, dtype=torch.float32), freqs)
        self.register_buffer("freqs_cis", torch.polar(torch.ones_like(angles), angles))

    def forward(self, seq_len: int) -> torch.Tensor:
        return self.freqs_cis[:seq_len]


class ALiBiBias(torch.nn.Module):
    """ALiBi 线性偏置"""

    def __init__(self, n_heads: int, max_seq_len: int = 512, alpha: float = 1.0):
        super().__init__()
        self.n_heads = n_heads
        self.alpha = alpha
        alibi_bias = self._generate(max_seq_len, n_heads, alpha)
        self.register_buffer("bias", alibi_bias)

    def _generate(self, seq_len: int, n_heads: int, alpha: float) -> torch.Tensor:
        positions = torch.arange(seq_len, dtype=torch.long)
        distances = torch.abs(positions.unsqueeze(1) - positions.unsqueeze(0))
        heads_log2_scale = torch.floor(torch.log2(torch.arange(1, n_heads + 1, dtype=torch.float32)))
        head_scales = alpha * torch.pow(2, -heads_log2_scale)
        return -head_scales.view(n_heads, 1, 1) * distances.view(1, seq_len, seq_len)

    def forward(self, seq_len: int) -> torch.Tensor:
        return self.bias[:, :seq_len, :seq_len]

```python
def verify_relative_position_invariance():
    """
    验证三种方法的相对位置不变性。
    相对位置不变性 = 注意力分数只依赖于 i-j，不依赖于 i 和 j 的绝对值。
    """
    seq_len = 32
    head_dim = 64
    n_heads = 4
    batch_size = 1

    # 使用固定的 QKV 以隔离位置编码的影响
    torch.manual_seed(42)
    q = torch.randn(batch_size, seq_len, n_heads, head_dim)
    k = torch.randn(batch_size, seq_len, n_heads, head_dim)

    print("=== 相对位置不变性验证 ===")
    print("检查注意力分数: score[i,j] == score[i+k, j+k] (same relative distance)")
    print()

    for method in ["RoPE", "ALiBi"]:
        if method == "RoPE":
            freqs_cis = precompute_freqs_cis(head_dim, seq_len)
            q_r, k_r = apply_rotary_pos_emb(q, k, freqs_cis)
        elif method == "ALiBi":
            alibi_bias = build_alibi_bias(seq_len, n_heads)

        scale = 1.0 / math.sqrt(head_dim)
        if method == "RoPE":
            scores = torch.einsum("bqhd,bkhd->bhqk", q_r, k_r) * scale
        else:
            scores_base = torch.einsum("bqhd,bkhd->bhqk", q, k) * scale
            scores = scores_base + alibi_bias.unsqueeze(0)

        # 检查 score[5,3] vs score[12,10] (相对距离都是 2)
        diff_2 = (scores[0, 0, 5, 3] - scores[0, 0, 12, 10]).abs().item()

        # 检查 score[5,0] vs score[20,15] (相对距离都是 5)
        diff_5 = (scores[0, 0, 5, 0] - scores[0, 0, 20, 15]).abs().item()

        print(f"{method}:")
        print(f"  |score[5,3] - score[12,10]| (dist=2): {diff_2:.8f}")
        print(f"  |score[5,0] - score[20,15]| (dist=5): {diff_5:.8f}")
        print()


verify_relative_position_invariance()

- **ALiBi**：两个 diff 都应该接近 0（偏置项只依赖于距离，不依赖绝对位置）

## 8. 常见误区与注意事项

### 8.1 误区 1：Sinusoidal PE 是"可学习的"还是"固定的"

**澄清**：原始 Transformer 论文中的 Sinusoidal PE 是**固定的**，不参与梯度更新。模型并没有为位置编码学习额外的参数。有些实现（如 BERT/GPT-2 早期版本）会将位置编码改为**可学习的嵌入**，但这已经不是原始的 Sinusoidal PE 了。

可学习的位置嵌入在训练效率上可能更优（模型可以直接学习位置表示），但失去了 Sinusoidal PE 的泛化能力和理论上"任意长度外推"的特性。

### 8.2 误区 2：RoPE 改变了 token 的语义向量

**澄清**：RoPE 作用在 $Q$ 和 $K$ 上，不改变 $V$。输出的 value 仍然来自 $V$，而 $V$ 是不包含位置信息的。因此，RoPE **不修改 token 的语义表示**，只是让 attention 机制在计算相似度时感知到位置信息。这与 Sinusoidal PE 加到输入 embedding 上有本质区别。

### 8.3 误区 3：ALiBi 的偏置系数是任意设计的

**澄清**：ALiBi 的头间衰减系数 $\frac{1}{2^{\lfloor \log_2(h+1) \rfloor}}$ 是一种**对数尺度的等比数列**。这意味着 head 0-1 的衰减最慢（系数=1），head 2-3 的衰减是前者的一半（系数=0.5），head 4-7 又是前者的四分之一（系数=0.25），依此类推。

这种设计的直觉是：head 数量越多，不同 head 能覆盖的衰减范围就越多样。如果有 8 个 head，从 1/1 到 1/8 的衰减率范围足以覆盖大多数语言的 attention 模式。

### 8.4 误区 4：位置编码只用在第一层

**澄清**：在原始 Transformer 中，位置编码是加到输入 embedding 上的，因此**所有层都能"看到"位置信息**。但不同层对位置信息的利用方式可能不同——浅层可能更多利用精确的绝对位置，深层可能更多利用通过多层 attention 堆叠形成的相对位置关系。

对于 RoPE，每个 attention 层都独立应用旋转，这保证了位置信息在每一层都被正确处理。

### 8.5 注意事项：RoPE 与 Flash Attention 的兼容细节

RoPE 本身与 Flash Attention 兼容，但需要注意以下几点：

1. **旋转必须在 Flash Attention 调用之前完成**：Flash Attention 的核心是分块计算 $S = QK^T$ 和 $P = \text{softmax}(S)$，然后分块聚合 $O = PV$。RoPE 的旋转是对 $Q$ 和 $K$ 的操作，必须在调用 Flash Attention 之前完成。

2. **不要在 Flash Attention 内部修改掩码**：ALiBi 的偏置通常通过修改掩码（mask）来实现，但 Flash Attention 的掩码格式有特定要求。正确做法是将偏置加到 $S = QK^T$ 上，而不是修改掩码。

3. **cuDNN Flash Attention 2+ 支持 RoPE**：主流实现（如 vLLM、Flash Attention 2+）已经内置了 RoPE 支持，在调用 flash attention 时传入旋转后的 Q/K 即可。

### 8.6 注意事项：推理时序列长度超出预计算范围

RoPE 的 `precompute_freqs_cis` 需要预计算到最大序列长度。如果推理时的序列长度超过了这个预计算范围，会导致 IndexError 或使用未经预训练的旋转角度。

**正确做法**：在初始化时将 `max_seq_len` 设置为模型可能用到的最大长度（考虑到 RoPE 的长度外推技术，可以设置得比较大，如 128K 或更高）。或者使用动态生成（但这会引入额外的计算开销）。

### 8.7 注意事项：ALiBi 与因果掩码的结合

ALiBi 的偏置项加在注意力分数上后，还需要与因果掩码（Causal Mask）结合。这里有一个常见错误：

```python
# 错误做法：偏置和掩码分开处理
scores = qk_scores + alibi_bias
scores = scores.masked_fill(torch.tril(torch.ones(seq_len, seq_len)) == 0, float("-inf"))

# 正确做法：先因果掩码，再加偏置（或者偏置自然包含在掩码中）
causal_mask = torch.tril(torch.ones(seq_len, seq_len), diagonal=0).bool()
scores = qk_scores.masked_fill(~causal_mask, float("-inf"))  # 先因果
scores = scores + alibi_bias  # 再加 ALiBi 偏置
```

因果掩码和 ALiBi 偏置是**独立的操作**，顺序不重要，但必须都应用。
