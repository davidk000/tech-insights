## 📌 今日课题
**长思考模型(o1/o3)：隐式推理/PRM/测试时计算 Scaling**

---

## 📖 完整讲解

## 原理

o1（OpenAI 2024）和 o3（2024 年底）是「测试时计算 Scaling」的代表——不再只是增大训练规模，而是在推理阶段分配更多计算资源。

### 传统 LLM 推理

输入 -> 一次前向传播 -> 输出 token

### o1/o3 推理

输入 -> 多次推理迭代（内部思考链）-> 输出 token

每个 token 生成时，模型会进行内部「思考」：评估当前回答的质量，考虑替代方案，可能推翻重来。

### PRM（Process Reward Model）

关键创新：用过程奖励模型（PRM）替代结果奖励模型（ORM）。

- ORM：只在最终答案评分
- PRM：在每个推理步骤评分，引导模型「一步步做对」

### Scaling Laws 的新方向

传统 Scaling Laws = 增大「训练时」计算量

o1/o3 = 增大「推理时」计算量

这意味着：不需要更大的模型，而是让模型花更多时间思考。

## 常见误区

1. "o1 比 GPT-4 基础能力更强" -> 错，o1 在纯知识问答上可能更差，优势在推理
2. "推理时计算 Scaling 会取代训练 Scaling" -> 两者互补，不是替代
3. "o1 只是在 CoT 外面包了一层" -> 关键差异是 PRM 的过程监督
