## 原理

Triton = 中间层 DSL，让用户写并行 kernel 而不碰 CUDA 细节。

CUDA 的痛点：手写 GPU 代码门槛极高，需要手动管理内存层级、线程协作、寄存器分配。

Triton 的核心：提供足够高层抽象，自动做 kernel 融合和自动调优。

| 概念 | 含义 | 类比 |
|:---|:---|:---|
| Grid | 整体并行规模 | GPU 总共开多少线程 |
| Block | 线程协作小组 | 一组工人共同处理一块数据 |
| Threads | 最小执行单元 | 实际干活的工人 |

```python
@triton.jit
def softmax_kernel(output_ptr, input_ptr, n_cols, BLOCK_SIZE: tl.constexpr):
    row_idx = tl.program_id(0)
    row_start_ptr = input_ptr + row_idx * n_cols
    col_offsets = tl.arange(0, BLOCK_SIZE)
    mask = col_offsets < n_cols
    vals = tl.load(input_ptr + row_start_ptr + col_offsets, mask=mask, other=float('-inf'))
    vals = vals - tl.max(vals, axis=0)
    numerator = tl.exp(vals)
    denominator = tl.sum(numerator, axis=0)
    result = numerator / denominator
    tl.store(output_ptr + row_start_ptr + col_offsets, result, mask=mask)
```

Triton 的价值：开发效率接近 Python，性能接近 CUDA。
