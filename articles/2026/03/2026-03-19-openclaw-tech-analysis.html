这是什么？

OpenClaw 是一个本地优先的个人 AI 助手框架，
让你在自有设备上运行私有的智能助手，
通过 WhatsApp、Telegram、Slack 等 12+ 渠道与你交互。

🏠
本地优先
完全本地运行，数据不出设备，隐私可控
🔌
多通道原生
12+ 即时通讯平台开箱即用，无需额外开发
🛠️
技能系统
Markdown + YAML 定义技能，ClawHub 生态扩展
3

架构概览

📱 WhatsApp · Telegram · Slack · Discord · Signal · WebChat
↓
🌐 Gateway (控制平面)
ws://127.0.0.1:18789 · 会话管理 · 通道管理 · 任务调度
↓
🤖 Pi Agent (运行时)
工具调用 · 模型交互 · 技能加载

采用 "Gateway + Agent" 分布式控制平面架构
单一 Gateway 控制所有消息通道，Agent 负责智能处理

4

核心组件

  • Gateway（网关） — WebSocket 控制平面，TypeBox 类型验证，设备配对系统
  • Agent Runtime — 基于 pi-mono，工作区隔离，会话 JSONL 存储
  • Channel Adapters — Baileys(WhatsApp)、grammY(Telegram)、Bolt(Slack) 等
  • Canvas Host — A2UI（Agent-to-User）可视化工作空间，独立端口 18793
  • Node System — iOS/macOS/Android 设备作为 Agent 能力延伸
  • Skills Platform — 三级加载优先级（工作区 > 本地 > 内置）
